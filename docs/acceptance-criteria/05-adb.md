# Критерии приёмки: 05 — ADB

**Практика:** [05 — ADB](../practices/05-adb.md)

Приложение FinControl поставляется из коробки. Для практик Sentry, AppMetrica и Firebase ключи подставляются в указанный файл: `lib/config/student_env.dart` (Sentry, AppMetrica) или конфиги Firebase в проекте. См. [STUDENT_ENV.md](../STUDENT_ENV.md).

## Обязательно

- [ ] FinControl установлен на эмулятор/устройство через `adb install` (или эквивалент).
- [ ] Выполнена команда `adb logcat` с фильтром по процессу или тегу `[FinControl]`; в логах видны записи приложения при действиях пользователя.
- [ ] Выполнено снятие скриншота (`adb exec-out screencap` или аналог) — файл получен и открывается.
- [ ] Выполнена очистка данных приложения (`adb shell pm clear <package>`); при следующем запуске приложение в начальном состоянии (например приветственный экран, пустой список).

## Желательно

- [ ] Выполнена запись видео с экрана (`adb shell screenrecord`) — файл mp4 скачан на ПК.
- [ ] Запущен monkey-тест (`adb shell monkey -p <package> -v 500`) — приложение не упало.
- [ ] Продемонстрировано управление разрешениями (grant/revoke) — например отзыв камеры и проверка поведения.
- [ ] Продемонстрирована беспроводная отладка (Wi-Fi) или управление авиарежимом для проверки offline-режима.

## Что показать на созвоне

Терминал с командами: установка APK, `adb logcat` с фильтром `[FinControl]`, скриншот, видео, monkey-тест, очистка данных. Подробнее: [exam-and-submission.md](../exam-and-submission.md#3-что-показать-и-что-сказать-на-экзамене-по-блокам).
