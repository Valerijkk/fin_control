# Критерии приёмки: 06 — Sentry

**Практика:** [06 — Sentry](../practices/06-sentry.md)

Приложение FinControl поставляется из коробки. Для Sentry DSN подставляется в указанный файл: `lib/config/student_env.dart` (константа `sentryDsn`). Реальный DSN не коммитить. См. [STUDENT_ENV.md](../STUDENT_ENV.md).

## Обязательно

- [ ] Создан проект в sentry.io (или использован существующий), получен DSN для Flutter.
- [ ] DSN подставлен в `lib/config/student_env.dart` (константа `sentryDsn`), Sentry инициализирован в `main.dart`.
- [ ] В настройках приложения при наличии DSN отображается кнопка «Тест Sentry» (или тестовое исключение вызвано иным способом).
- [ ] После нажатия «Тест Sentry» (или краша) событие/краш появилось в дашборде Sentry (с учётом задержки отправки при перезапуске).

## Желательно (безопасность)

- [ ] Ученик понимает: реальный DSN не коммитится в репозиторий; при push в `student_env.dart` оставлены пустые строки (см. [STUDENT_ENV.md](../STUDENT_ENV.md)).

## Что показать на созвоне

Кнопка «Тест Sentry» в настройках FinControl; событие/исключение в кабинете Sentry. Подробнее: [exam-and-submission.md](../exam-and-submission.md#3-что-показать-и-что-сказать-на-экзамене-по-блокам).
