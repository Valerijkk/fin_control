# Критерии приёмки: 01 — Charles Proxy

**Практика:** [01 — Charles Proxy](../practices/01-charles.md)

Приложение FinControl поставляется из коробки. Для практик Sentry, AppMetrica и Firebase ключи подставляются в указанный файл: `lib/config/student_env.dart` (Sentry, AppMetrica) или конфиги Firebase в проекте. См. [STUDENT_ENV.md](../STUDENT_ENV.md).

## Обязательно

- [ ] Charles установлен и запущен, прокси на устройстве/эмуляторе настроен (хост + порт 8888).
- [ ] Сертификат Charles установлен на устройстве и помечен как доверенный (для HTTPS).
- [ ] В Charles включён SSL Proxying для хостов API курсов (или *:443).
- [ ] Приложение FinControl открыто, выполнен переход на экран с загрузкой курсов (Обменник или главный с карточкой курсов).
- [ ] В Charles виден хотя бы один запрос к `api.exchangerate.host` или `open.er-api.com`.
- [ ] Ответ на запрос — HTTP 200, в теле JSON с полями курсов (rates или аналог).

## Желательно

- [ ] Ученик продемонстрировал **Breakpoint**: остановил ответ API, изменил курс валюты в JSON, показал изменённый курс в приложении.
- [ ] Ученик продемонстрировал **Map Local**: подставил локальный JSON-файл вместо ответа API.
- [ ] Ученик продемонстрировал **Throttling**: включил имитацию медленной сети и показал поведение приложения (таймаут, кэш).
- [ ] Понимание: перехват HTTPS с установкой сертификата Charles — только для учебной среды; не использовать на устройствах с чувствительными данными (см. [practices/01-charles.md](../practices/01-charles.md), блок «Безопасность»).

## Что показать на созвоне

Charles с записью трафика; в Charles виден запрос к API курсов при открытии Обменника в FinControl. Подробнее: [exam-and-submission.md](../exam-and-submission.md#3-что-показать-и-что-сказать-на-экзамене-по-блокам).
