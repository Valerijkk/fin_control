# 5 багов для задания под звёздочкой (только для экзаменатора)

Ученики не должны видеть этот файл. Ниже — перечень специально оставленных багов разной сложности.

---

## Баг 1 (лёгкий) — Обменник: нулевая сумма

**Где:** `lib/ui/screens/exchange_screen.dart`, метод `_performExchange`.

**Суть:** Валидация проверяет только `amount < 0`, но не `amount == 0`. Можно нажать «Обменять» при сумме 0 и сохранить операцию с нулевыми значениями в историю.

**Как воспроизвести:** Обменник → ввести 0 → Обменять. В истории появляется запись «0 RUB → 0 USD» (или другая пара).

**Исправление:** В условии заменить `amount < 0` на `amount <= 0` (или явно отсекать 0).

---

## Баг 2 (лёгкий) — Настройки: «Очистить все данные»

**Где:** Настройки → пункт «Очистить все данные»; логика в `state/app_state.dart` (clearAll) и вызов из `lib/ui/screens/settings_screen.dart`.

**Суть:** Заголовок говорит «Очистить все данные», но очищаются только записи расходов/доходов. История обмена и данные портфеля (баланс, позиции, сделки) не удаляются.

**Как воспроизвести:** Создать операции обмена и позиции в портфеле → Настройки → Очистить все данные → подтвердить. Расходы пропадут, история обмена и портфель останутся.

**Исправление:** Либо очищать также exchange_operations и portfolio (через репозитории), либо изменить заголовок/подзаголовок на «Очистить только расходы и доходы».

---

## Баг 3 (средний) — Портфель: деление на ноль при нулевом курсе

**Где:** `lib/ui/screens/portfolio_screen.dart`, метод `_rateToBase`.

**Суть:** Для USD и EUR возвращается `1 / _rates!.usd` и `1 / _rates!.eur` без проверки на 0. Если API (или мок) вернёт курс 0, получится деление на ноль (Infinity в double) и некорректные расчёты или отображение.

**Как воспроизвести:** Сложно без подмены API/ответа. Можно временно в `rates_api.dart` вернуть `Rates(0, 0, ...)` и открыть портфель / попробовать купить — возможны Infinity в UI или сбой логики.

**Исправление:** В `_rateToBase` перед возвратом проверять `_rates!.usd != 0` и `_rates!.eur != 0`, при 0 возвращать `null` или не использовать такой курс.

---

## Баг 4 (средний) — Запись расхода/дохода с нулевой суммой

**Где:** `lib/ui/screens/add_edit_screen.dart`, валидатор поля «Сумма».

**Суть:** В валидаторе проверяется `x < 0`, но не `x == 0`. Можно сохранить запись с суммой 0 (расход или доход с нулём).

**Как воспроизвести:** Добавить запись → Сумма «0», название любое → Сохранить. Запись с 0 появляется в списке и в статистике.

**Исправление:** В валидаторе заменить на `x == null || x <= 0` и сообщение «Введите сумму > 0».

---

## Баг 5 (сложный) — Обменник: setState после dispose

**Где:** `lib/ui/screens/exchange_screen.dart`, метод `_loadRates`, блок `catch`.

**Суть:** После `await RatesApi.fetch()` есть проверка `if (mounted)` перед setState, но в блоке `catch` setState вызывается **без** проверки `mounted`. Если пользователь быстро уходит с экрана обменника во время загрузки курсов и запрос завершается ошибкой, вызов setState произойдёт после dispose виджета.

**Как воспроизвести:** Открыть Обменник → сразу переключиться на другой экран (или выйти) до завершения загрузки курсов; при этом симулировать ошибку сети (например отключить интернет или подменить API на недоступный). В debug-сборке может появиться предупреждение/ошибка setState after dispose.

**Исправление:** В блоке catch перед setState добавить проверку `if (mounted) setState(...)`.
