# Тест-кейсы и автотесты FinControl

Приложение **одно**, запускается **из коробки** (`flutter run`). Для проверки Sentry, AppMetrica и Firebase достаточно подставить ключи в указанный файл: **Sentry и AppMetrica** — в [`lib/config/student_env.dart`](../../lib/config/student_env.dart) (см. [docs/STUDENT_ENV.md](../STUDENT_ENV.md)); **Firebase** — по инструкциям в [docs/practices/](../practices/) (конфиги проекта).

В репозитории есть **шаблоны** тестов в `test/` (unit- и widget-тесты); остальной функционал покрыт ручными тест-кейсами ниже. При необходимости новые автотесты добавляются по тем же шаблонам.

## 2. Ручные тест-кейсы по фичам

### 2.1 Учёт расходов (Home, Add/Edit)

| ID | Название | Предусловия | Шаги | Ожидаемый результат |
|----|----------|-------------|------|---------------------|
| M-1.1 | Добавление расхода | Приложение открыто, Shell → Список | Нажать «+», ввести сумму, название, категорию, сохранить | Запись появляется в списке |
| M-1.2 | Редактирование записи | В списке есть запись | Тап по записи, изменить название/сумму, сохранить | Изменения отображаются в списке |
| M-1.3 | Удаление с UNDO | В списке есть запись | Свайп влево по записи, в снэкбаре нажать ОТМЕНА | Запись восстанавливается |
| M-1.4 | Фильтр по категории | В списке записи разных категорий | Выбрать чип «Еда» | Отображаются только записи категории Еда |
| M-1.5 | Быстрое добавление | Список открыт | Тап по иконке быстрой записи, ввести сумму, Добавить | Запись с дефолтным названием в списке |

### 2.2 Обменник

| ID | Название | Предусловия | Шаги | Ожидаемый результат |
|----|----------|-------------|------|---------------------|
| M-2.0 | Отображение курсов на экране | Интернет включён | Открыть вкладку «Обменник» | Отображаются курсы (RUB/USD/EUR или карточка с курсами); при наличии — источник/время обновления |
| M-2.1 | Расчёт по курсу | Интернет включён, Обменник открыт | Выбрать валюты «из» и «в», ввести сумму «из» | Сумма «в» пересчитывается по актуальному курсу |
| M-2.2 | Сохранение операции | Курсы загружены | Ввести сумму > 0, нажать «Обменять» | Операция появляется в истории обмена |
| M-2.3 | Валидация суммы 0 | Обменник открыт | Ввести 0, нажать «Сохранить операцию» | Кнопка неактивна или отображается сообщение об ошибке (уточнение — в [задании со звёздочкой](../acceptance-criteria/bugs-dlya-ekzamenatora.md)) |
| M-2.4 | Ошибка сети | Выключить сеть / неверный прокси | Открыть Обменник | Сообщение об ошибке, приложение не падает |
| M-2.5 | Оповещение по курсу | Курсы загружены | «Добавить оповещение» → пара валют, целевой курс, условие (≥/≤) → Добавить | Оповещение в списке; при следующем обновлении курсов при срабатывании — SnackBar и отметка «Сработало» |
| M-2.6 | Отложенный обмен (limit) | Курсы загружены | «Добавить отложенный обмен» → сумма, пара, курс исполнения, условие → Добавить | Ордер в списке «Ожидание»; при достижении курса — автоисполнение, запись в истории обменов, SnackBar |

### 2.3 Портфель

| ID | Название | Предусловия | Шаги | Ожидаемый результат |
|----|----------|-------------|------|---------------------|
| M-3.1 | Отображение баланса | Портфель открыт | — | Отображается баланс в базовой валюте |
| M-3.2 | Покупка актива | Баланс > 0 | Купить → валюта, сумма → подтвердить | Баланс уменьшается, в активах появляется/обновляется позиция |
| M-3.3 | Продажа актива | Есть позиция по валюте | Продать → указать сумму/количество → подтвердить | Позиция уменьшается, баланс увеличивается |
| M-3.4 | История сделок | Есть хотя бы одна сделка | Прокрутить до блока истории | Отображаются тип, валюта, сумма, дата |
| M-3.5 | Доходность (PnL) и ROI | Есть позиции и/или сделки, курсы загружены | Открыть Портфель, прокрутить до блока «Доходность (PnL)» | Отображаются общая стоимость, cost basis, нереализ. PnL, реализ. PnL, ROI % |
| M-3.6 | Аллокация активов | Есть хотя бы одна позиция, курсы загружены | Открыть Портфель | Блок «Аллокация активов» с долями в % и полосками по валютам/балансу |

### 2.4 Статистика

| ID | Название | Предусловия | Шаги | Ожидаемый результат |
|----|----------|-------------|------|---------------------|
| M-4.1 | Итоги расходов/доходов | Есть записи расходов и доходов | Открыть Статистику | Отображаются «Всего расходов» и при необходимости доходы |
| M-4.2 | Разбивка по категориям | Есть записи в разных категориях | Открыть Статистику | Видны категории и суммы/полоски по ним |

### 2.5 Настройки

| ID | Название | Предусловия | Шаги | Ожидаемый результат |
|----|----------|-------------|------|---------------------|
| M-5.1 | Переключение темы | Настройки открыты | Переключить тёмную/светлую тему | Тема меняется сразу и сохраняется после перезапуска |
| M-5.2 | Очистить кэш курсов | Обменник ранее загружал курсы | Настройки → Очистить кэш курсов | При следующем открытии Обменника курсы запрашиваются заново |
| M-5.3 | Очистить все данные | Есть расходы и/или история обмена/портфеля | Настройки → «Очистить все данные», подтвердить | Все локальные данные удалены (расходы, обмен, портфель); уточнение границ — в [задании со звёздочкой](../acceptance-criteria/bugs-dlya-ekzamenatora.md) |

### 2.6 Приветствие и навигация

| ID | Название | Предусловия | Шаги | Ожидаемый результат |
|----|----------|-------------|------|---------------------|
| M-6.1 | Первый запуск | Чистая установка | Запустить приложение | Отображается приветственный экран, кнопка «Начать» |
| M-6.2 | Переход в приложение | Приветствие отображается | Нажать «Начать» | Отображается Shell с нижней навигацией |
| M-6.3 | Переключение вкладок | Shell открыт | Тап по вкладкам: Обменник, Портфель, Статистика | Соответствующий экран отображается |
| M-6.4 | Открытие настроек | Shell открыт | Тап по иконке/пункту «Настройки» | Отображается экран настроек (тема, кэш, очистка данных) |

### 2.7 Цель накопления (главный экран)

| ID | Название | Предусловия | Шаги | Ожидаемый результат |
|----|----------|-------------|------|---------------------|
| M-7.1 | Добавить цель | Главный экран открыт | В карточке «Цель накопления» нажать «Добавить цель» → название, целевая сумма → Добавить | Цель появляется с прогрессом 0% |
| M-7.2 | Пополнить цель | Есть цель | В карточке цели нажать «Пополнить» → ввести сумму → Пополнить | Текущая сумма и % прогресса увеличиваются |
| M-7.3 | Синхронизация с портфелем | Есть цель и баланс в Портфеле | Нажать «Из портфеля» | Текущая сумма цели устанавливается равной балансу портфеля |
