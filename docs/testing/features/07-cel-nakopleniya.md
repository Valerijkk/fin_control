# Фича: Цель накопления

## 1. Бизнес-требования

- Пользователь может задать цель накопления (название, целевая сумма в базовой валюте) и отслеживать прогресс.
- Текущая сумма цели пополняется вручную или подставляется из баланса портфеля («Из портфеля»).

## 2. Функциональные требования

| ID | Требование | Приоритет |
|----|------------|-----------|
| FR-7.1 | Карточка «Цель накопления» на главном экране (под карточкой курсов) | Высокий |
| FR-7.2 | Добавление цели: название, целевая сумма (валюта из настроек портфеля) | Высокий |
| FR-7.3 | Отображение прогресса: текущая сумма / цель, процент, прогресс-бар | Высокий |
| FR-7.4 | Кнопка «Пополнить»: ввод суммы, увеличение текущей суммы цели | Высокий |
| FR-7.5 | Кнопка «Из портфеля»: подстановка текущего баланса портфеля в текущую сумму цели | Средний |

## 3. Схема БД

Таблица **savings_goals** (SQLite):

| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT |
| title | TEXT | NOT NULL |
| target_amount | REAL | NOT NULL |
| base_currency | TEXT | NOT NULL |
| current_amount | REAL | NOT NULL DEFAULT 0 |
| deadline | INTEGER | NULL, опционально |
| created_at | INTEGER | NOT NULL |

## 4. Ожидаемое поведение UI

- **Кнопки:** «Добавить цель», «Пополнить», «Из портфеля».
- **Обратная связь:** после добавления цели или пополнения — обновление прогресса и процента на карточке.
