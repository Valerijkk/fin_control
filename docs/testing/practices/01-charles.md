# Практика 01: Charles Proxy

## Технология

**Charles Proxy** — прокси-сервер для перехвата и анализа HTTP/HTTPS трафика с мобильных устройств и десктопа. Тестировщики используют его для проверки запросов к API, подмены ответов (breakpoints), эмуляции медленной сети (throttling).

## В проекте FinControl

- Все запросы к API курсов валют идут по **HTTPS** (exchangerate.host, open.er-api.com).
- Запросы выполняются при открытии экрана **Обменник** и при работе с **Портфелем** (курсы для расчёта).
- В запросах заданы User-Agent и заголовки, чтобы трафик однозначно идентифицировать в Charles.

## Задание

1. Установить Charles, настроить прокси на устройстве/эмуляторе (порт 8888).
2. Установить и доверить сертификат Charles на устройстве для расшифровки HTTPS.
3. Включить SSL Proxying для нужных хостов.
4. Запустить FinControl, открыть Обменник (или экран с загрузкой курсов).
5. Убедиться, что в Charles виден запрос к API курсов и ответ 200 с JSON (rates, date).
