# Практика 12: Firebase Analytics

## Технология

**Firebase Analytics** — события и воронки: открытие экранов, действия пользователя. Тестировщику нужно уметь настраивать события в коде и проверять их в отчётах Firebase (DebugView в режиме отладки).

## В проекте FinControl

- После 00-firebase-setup подключается `firebase_analytics`. Логирование экранов (Обменник, Портфель, Список и т.д.) и при необходимости кастомных событий (например «обмен выполнен»).

## Задание

1. Подключить Firebase Analytics в проекте.
2. Добавить логирование открытия экранов (screen_view) и при необходимости ключевых действий.
3. Запустить приложение в debug, выполнить сценарий (переходы по экранам).
4. Проверить появление событий в Firebase Console → Analytics → DebugView.

**Полная инструкция:** [docs/practices/12-firebase-analytics.md](../../practices/12-firebase-analytics.md).
