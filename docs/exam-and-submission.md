## 🎯 1. Что хотят в целом

- **Проект из коробки:** всё уже настроено. Для метрик — вставь токены в [STUDENT_ENV.md](STUDENT_ENV.md), для Firebase — следуй инструкции в [10-firebase.md](practices/10-firebase.md).
- **Выполни практики** по порядку из [practices-step-by-step.md](practices-step-by-step.md).
- **Покажи результат:** на созвоне продемонстрируй, что всё работает, и кратко расскажи, что делал.
- **Подтверди понимание:** не просто «нажал кнопку», а «настроил прокси на устройстве, установил сертификат Charles, открыл приложение — в Charles виден запрос к API курсов».

> 📌 От тебя **не** требуют идеальных скриншотов или отчётов на 10 страниц. Нужны: **выполненная практика + умение показать и объяснить**.

## 📚 2. Как готовиться к сдаче

1. **Проходи практики по порядку** из [practices-step-by-step.md](practices-step-by-step.md): 00-getting-started → 01–09 → 10-firebase.md
2. **Используй пошаговые инструкции** в [practices/](practices/) — там расписано, что нажимать и что должно получиться. При ошибках — [FAQ.md](FAQ.md)
3. **Сверяйся с чек-листами** в [acceptance-criteria/](acceptance-criteria/) — по каждой практике файл с критериями приёмки; отметь все пункты «Обязательно» перед сдачей
4. **За день до созвона** проверь:
   - приложение / Charles / Sentry / AppMetrica открываются и работают
   - можешь за 2–5 минут показать результат
   - можешь одним-двумя предложениями объяснить, что делал и чему научился
5. **Если сдаёшь удалённо** — проверь Share Screen: эмулятор или телефон, Charles/Proxyman, браузер с Sentry/AppMetrica. Закрой лишние вкладки

## 🗣 3. Как обычно проходит созвон

| Параметр | Описание |
|----------|----------|
| **Формат** | Экзаменатор просит по очереди показать каждую практику. На одну — **2–5 минут** |
| **Что показывать** | То, что описано в таблицах ниже — без длинных объяснений, по делу |
| **Вопросы** | Экзаменатор может спросить: «Зачем нужен сертификат?», «Что такое DSN?» — ответы в разделе [Типичные вопросы](#-9-типичные-вопросы-экзаменатора) |
| **Если что-то не работает** | Не паникуй. Перезапусти приложение / обнови страницу. Если среда сломана — кратко опиши, что делал |

### 🔥 Firebase (10 — единая практика)

> Firebase теперь — **одна практика** [10-firebase.md](practices/10-firebase.md), которая включает все модули: setup, Crashlytics, FCM, Analytics, Remote Config, Performance, In-App Messaging.

| Модуль | Что показать | Что сказать (пример) |
|--------|-------------|----------------------|
| **Setup** | В консоли Firebase — приложения Android/iOS; в проекте `google-services.json` / `GoogleService-Info.plist` | «Создал проект Firebase, добавил приложения, скачал конфиги — приложение подключается без ошибок.» |
| **Crashlytics** | Тестовый краш → отчёт в Crashlytics | «Вызвал тестовый краш — отчёт в консоли Firebase.» |
| **FCM** | Токен в логах; тестовое push из консоли | «Приложение получает токен; отправил push из консоли — пришло.» |
| **Analytics** | События (открытие экранов, обмен) → Firebase Analytics | «Прошёл сценарии — в Firebase вижу события.» |
| **Remote Config** | Параметр в консоли → значение в приложении | «Добавил параметр, делаю fetch/activate — значение подставляется.» |
| **Performance** | Трейсы в Firebase Performance | «Походил по экранам — в консоли появились трейсы.» |
| **In-App Messaging** | Кампания → сообщение в приложении | «Создал кампанию с триггером — сообщение отображается.» |

## 🧐 9. Типичные вопросы экзаменатора

> Вопросы согласованы с [acceptance-criteria/](acceptance-criteria/). Ответы не нужно заучивать — достаточно понимать суть.

---

### 📋 Базовые вопросы

| Вопрос | О чём ответить кратко |
|--------|----------------------|
| **Что ты делал в этой практике?** | Перечисли ключевые шаги: «Создал эмулятор, установил приложение, в Logcat отфильтровал по пакету — вижу логи.» |
| **Зачем нужен Charles / Proxyman?** | Перехватывать HTTP/HTTPS-трафик: запросы к API, ответы. Для отладки и проверки. |
| **Почему нужен сертификат на устройстве?** | Расшифровать HTTPS. Без доверенного сертификата — видно только факт соединения, не содержимое. |
| **Что такое DSN в Sentry?** | Адрес (URL) проекта в Sentry, по которому приложение отправляет события. |
| **Зачем AppMetrica?** | Аналитика: сессии, устройства, поведение пользователей — без доступа к устройствам. |
| **Что показать по Firebase?** | Проект в консоли, приложение с правильным package name, конфиги в проекте, `Firebase.initializeApp()`, для модулей — соответствующий сервис в консоли и в приложении. |
| **Что делать, если на созвоне что-то не открылось?** | Перезапустить / обновить страницу. Если среда сломана — описать, что делал. |
| **Где вставляются токены Sentry и AppMetrica?** | Только в `lib/config/student_env.dart`. Firebase — конфиги по [10-firebase.md](practices/10-firebase.md). |

---

### 🌟 Продвинутые вопросы (для отличников)

| Вопрос | О чём ответить кратко |
|--------|----------------------|
| **Как подменить ответ API в Charles?** | Breakpoints: остановить ответ, изменить JSON. Или Map Local: подготовить файл с ответом. |
| **Что такое Throttling?** | Имитация медленной сети. Проверить: показывает ли приложение индикатор загрузки, не зависает ли UI. |
| **Что такое monkey-тестирование?** | Случайные действия через `adb shell monkey`. Ищет краши и ANR от непредсказуемых действий. |
| **Как найти утечку памяти?** | Android Studio: Memory Profiler → GC → если память не уменьшается — утечка. Xcode: Memory Graph Debugger. |
| **Что такое breadcrumbs в Sentry?** | Цепочка действий пользователя перед ошибкой. Помогает воспроизвести баг. |
| **В чём разница Crashlytics и Sentry?** | Оба отслеживают краши. Crashlytics — часть Firebase (Google). Sentry — независимый, с более гибкой настройкой. |
| **Зачем Remote Config?** | Изменение поведения без обновления: фичефлаги, rollout, A/B-тесты. |
| **Как отслеживать регрессии производительности?** | Firebase Performance: кастомные трейсы, сравнение метрик между версиями. |
