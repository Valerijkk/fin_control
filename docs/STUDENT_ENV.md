# Переменные для сборки под ученика (индивидуальный запуск)

Все ключи и DSN, которые нужно вставить во время прохождения практик, задаются **в одном файле**. Нажми ссылку — откроется файл, вставь значения в пустые строки и сохрани:

→ **[Открыть файл с переменными (student_env.dart)](../lib/config/student_env.dart)**

В начале файла капсом подписано, что куда вставлять и для какой практики. Можно заполнять поэтапно: после практики 06 вставил только DSN → перезапуск → потом после практики 07 вставил API Key AppMetrica → перезапуск. Приложение при любом наборе переменных (ни одной, одна, обе) работает корректно.

---

## Какие переменные заполнять по практикам

| Практика | Переменная в `student_env.dart` | Откуда взять значение |
|----------|---------------------------------|------------------------|
| **06 Sentry** | `sentryDsn` | Кабинет [sentry.io](https://sentry.io) → проект → Settings → Client Keys (DSN). Формат: `https://xxxx@xxxx.ingest.sentry.io/xxxx` |
| **07 AppMetrica** | `appMetricaApiKey` | Кабинет [appmetrica.io](https://appmetrica.io) → приложение → настройки → API Key |
| **00 Firebase, 10–15** | не в этом файле | Свой проект Firebase; файлы `google-services.json` и `GoogleService-Info.plist` клади в проект по инструкции — см. [practices/00-firebase-setup.md](practices/00-firebase-setup.md) |

Вставь значение **в пустую строку** между кавычками, например: `const String sentryDsn = 'https://xxx@xxx.ingest.sentry.io/xxx';`  
После изменения сохрани файл и перезапусти приложение (`flutter run`).

---

## Режимы запуска

- **Нет переменных или заполнены не все**  
  Приложение собирается и работает: локальная БД, список расходов, обменник, портфель, акции. Sentry и AppMetrica не инициализируются — кнопка «Тест Sentry» в настройках не появится, события в кабинеты не уходят. Удобно для начала и практик 01–05 (Charles, ADB и т.д.).

- **Заполнены нужные переменные**  
  Полноценный запуск по выбранным практикам: при инициализации подставляются твои DSN/API Key, телеметрия и аналитика работают. **Поэтапное выполнение учтено:** можно вставить только DSN (для практики 06), перезапустить — Sentry заработает, AppMetrica не трогаешь; потом добавить API Key (для практики 07), перезапустить — оба работают. Приложение корректно работает при любом наборе: ни одной переменной, только одна, или обе.
