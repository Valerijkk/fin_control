# Переменные для сборки под ученика (индивидуальный запуск)

Проект настроен **из коробки**. Для метрик и Firebase достаточно вставить токены в указанные места и следовать инструкциям — **после подстановки всё работает**.

---

## Единственное место для токенов Sentry и AppMetrica

**Токены Sentry (DSN) и AppMetrica (API Key)** вставляются **только** в один файл в коде:

→ **[lib/config/student_env.dart](../lib/config/student_env.dart)**

В начале файла капсом подписано, что куда вставлять и для какой практики. Других мест в проекте для этих ключей нет — не дублируй их в других файлах. Можно заполнять поэтапно: после практики 06 вставил только DSN → перезапуск → потом после практики 07 вставил API Key AppMetrica → перезапуск. Приложение при любом наборе (ни одной, одна, обе) работает корректно.

---

## Firebase: конфиги по инструкции

Для практик **00 Firebase setup** и **10–15** (Crashlytics, FCM, Analytics и др.) используются **конфиги Firebase**, а не `student_env.dart`. Файлы `google-services.json` (Android) и `GoogleService-Info.plist` (iOS) кладутся в проект **по инструкции** в [practices/00-firebase-setup.md](practices/00-firebase-setup.md). После добавления своих конфигов и вызова `Firebase.initializeApp()` в приложении всё работает без дополнительной подстановки токенов в код.

## Какие переменные заполнять по практикам

| Практика | Переменная в `student_env.dart` | Откуда взять значение |
|----------|---------------------------------|------------------------|
| **06 Sentry** | `sentryDsn` | Кабинет [sentry.io](https://sentry.io) → проект → Settings → Client Keys (DSN). Формат: `https://xxxx@xxxx.ingest.sentry.io/xxxx` |
| **07 AppMetrica** | `appMetricaApiKey` | Кабинет [appmetrica.io](https://appmetrica.io) → приложение → настройки → API Key |
| **00 Firebase, 10–15** | не в этом файле | Свой проект Firebase; файлы `google-services.json` и `GoogleService-Info.plist` клади в проект по инструкции — см. [practices/00-firebase-setup.md](practices/00-firebase-setup.md) |

Вставь значение **в пустую строку** между кавычками, например: `const String sentryDsn = 'https://xxx@xxx.ingest.sentry.io/xxx';`  
После изменения сохрани файл и перезапусти приложение (`flutter run`).

## Безопасность ключей

- **Не коммить реальные ключи в репозиторий.** Файл `lib/config/student_env.dart` предназначен для локальной разработки и практик. При отправке кода (push) оставляй в константах **пустые строки** `''` или не включай в коммит заполненные значения — иначе DSN и API Key попадут в историю Git и могут быть скомпрометированы.
- **Один источник:** все ключи для практик (Sentry, AppMetrica) задаются только в `student_env.dart`; дублировать их в других файлах или хардкодить в коде не следует.
- **Для продакшена:** в реальных проектах секреты выносят в переменные окружения или защищённые конфиги (не в репозитории); в рамках учебного проекта достаточно не коммитить заполненный `student_env.dart`.

## Режимы запуска

- **Нет переменных или заполнены не все**  
  Приложение собирается и работает: локальная БД, список расходов, обменник, портфель, акции. Sentry и AppMetrica не инициализируются — кнопка «Тест Sentry» в настройках не появится, события в кабинеты не уходят. Удобно для начала и практик 01–05 (Charles, ADB и т.д.).

- **Заполнены нужные переменные**  
  Полноценный запуск по выбранным практикам: при инициализации подставляются твои DSN/API Key, телеметрия и аналитика работают. **Поэтапное выполнение учтено:** можно вставить только DSN (для практики 06), перезапустить — Sentry заработает, AppMetrica не трогаешь; потом добавить API Key (для практики 07), перезапустить — оба работают. Приложение корректно работает при любом наборе: ни одной переменной, только одна, или обе.
