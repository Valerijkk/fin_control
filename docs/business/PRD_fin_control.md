# PRD: FinControl — учебная инвестиционная платформа и обменник

**Единая терминология:** во всём проекте под FinControl понимается **одно приложение** (мобильное на Flutter; веб-сборка опциональна для практик по сниффу трафика), используемое для всех практик — Charles, Proxyman, Android Studio, Xcode, ADB, Sentry, AppMetrica, TestFlight, Android-дистрибуция, Firebase.

---

## 1. Цель продукта

**FinControl** — учебный pet-project для тестировщиков: одно и то же приложение (мобильное и при необходимости веб) на Flutter, объединяющее:

- **Учёт расходов/доходов** (текущий функционал)
- **Обменник валют** — калькулятор обмена по актуальным курсам, история операций
- **Мини-инвестиционная платформа** — виртуальный портфель (покупка/продажа «активов» по курсам валют), без реальных денег

Все сетевые запросы (курсы валют, при необходимости — свой API) проходят по HTTP/HTTPS, чтобы на базе этого же приложения отрабатывать практики: **Charles**, **Proxyman**, **Android Studio**, **Xcode**, **ADB**, **Sentry**, **AppMetrica**, **Firebase** (Crashlytics, FCM, Analytics, Remote Config, Performance, In-App Messaging), **TestFlight**, Android-дистрибуция.

---

## 2. Роли пользователей

- **Ученик (тестировщик)** — единственная роль: использует **одно и то же приложение** FinControl для учёта расходов/доходов, обменника и мини-портфеля, параллельно отрабатывая практики по Charles, Proxyman, Android Studio, Xcode, ADB, Sentry, AppMetrica, Firebase, TestFlight и Android-дистрибуции.

---

## 3. Функциональные требования

### 3.1 Учёт расходов/доходов (существующее)

- Запись расходов/доходов: сумма, название, категория, дата, фото чека
- Категории: встроенные + пользовательские (персист в shared_preferences)
- Список с фильтрами (категория, период, поиск), свайп-удаление, UNDO
- Статистика по категориям и периодам
- Тёмная тема (Material 3)
- Локальное хранение: SQLite (sqflite)

### 3.2 Обменник валют

- Экран «Обменник»: выбор валюты «из» и «в», ввод суммы, расчёт по актуальному курсу
- Курсы загружаются с публичных API (exchangerate.host, open.er-api.com — уже есть в проекте), кэш в shared_preferences
- Отображение курсов на экране (RUB/USD/EUR и при необходимости другие пары)
- **История операций обмена** (локально в SQLite): дата, сумма из, валюта из, сумма в, валюта в, курс на момент операции — для практики тестирования данных и сценариев

### 3.3 Мини-инвестиции

- Экран «Портфель»: виртуальный баланс в выбранной валюте (RUB/USD/EUR), список «активов»
- «Активы» — упрощённо: валютные пары (например «USD», «EUR»). Покупка/продажа по текущему курсу из API
- Локальное хранение портфеля и сделок в SQLite (без реальных денег)
- Отображение текущей стоимости портфеля и истории сделок

### 3.4 Настройки и инфраструктура

- Настройки: тема, базовая валюта для отображения, очистка кэша курсов
- Все запросы к API курсов — по HTTPS с явным User-Agent и заголовками, чтобы трафик был виден в Charles/Proxyman при настройке прокси на устройстве/эмуляторе

## 4. Нефункциональные требования

### 4.1 Платформы

- **Android** — минимум API 21, поддержка эмулятора и реального устройства
- **iOS** — поддержка симулятора и реального устройства (после добавления ios/)
- **Web** (опционально) — для практики сниффа в браузере через Charles/Proxyman

### 4.2 Инструменты для практик тестировщиков

Одно и то же приложение FinControl (мобильное и при необходимости веб) должно позволять отрабатывать практики:

| Инструмент | Что отрабатывается |
|------------|--------------------|
| **Charles** | Перехват HTTP/HTTPS трафика (HTTPS с доверенным сертификатом Charles) |
| **Proxyman** | Снифф трафика на Android/iOS |
| **Android Studio** | Установка в эмуляторе, настройка logcat |
| **Xcode** | Установка в симуляторе iOS, логи в Console |
| **ADB** | Установка, логи, скриншоты, очистка данных (мануал на базе FinControl) |
| **Sentry** | Подключение SDK, тестовые события и краши |
| **AppMetrica** | Заведение приложения, сессии и устройства |
| **TestFlight** | Сборка и публикация iOS-приложения |
| **Android-дистрибуция** | Сборка, подпись, Google Play Internal testing или Firebase App Distribution |
| **Firebase** | Проект создан, приложение подключено (iOS и/или Android); практики по модулям: Crashlytics, FCM, Analytics, Remote Config, Performance Monitoring, In-App Messaging |

### 4.3 UX-критерии (кратко)

- **Понятные подписи:** кнопки и поля имеют однозначные подписи (например «Сохранить», «Обменять», «Сумма»); диалоги подтверждения — ясные формулировки («Удалить запись?», «Подтвердите»).
- **Обратная связь при действиях:** после сохранения, удаления или операции — SnackBar или возврат на список с обновлёнными данными; при загрузке данных с сети — индикатор или состояние «офлайн»/ошибка с возможностью повторить.
- **Единообразие:** основные действия — одна и та же кнопка (FilledButton/PrimaryButton); настройки доступны с каждого экрана через иконку в AppBar; пустые состояния — единый стиль сообщений («Нет записей», «Пока нет операций»).

Детали по экранам (кнопки, навигация, ошибки, загрузка) — в [docs/testing/features](../testing/features/) (раздел «Ожидаемое поведение UI» в каждом файле фичи).

### 4.4 Документация для учеников

- Практики (пошаговые инструкции) по каждой технологии: что настроить, что проверить, какие сценарии тестировать на базе **одного и того же приложения** FinControl. Практики размещаются в `docs/practices`.

## 5. Ограничения и допущения

- Нет бэкенда в репозитории: курсы валют — только публичные API; портфель и обмен — локально в SQLite
- При необходимости демо «своего API» для сниффа можно добавить один минимальный сервер (например Node/Express или Dart shelf) с одним эндпоинтом — по решению команды
- Регистрация/авторизация не требуются (однопользовательское локальное приложение)
- Практики по Firebase/AppMetrica/Sentry предполагают наличие аккаунтов; в документации указать, что нужны тестовые проекты

## 6. Критерии готовности

Во всех пунктах речь о **одном приложении** FinControl (мобильное; веб — опционально).

1. **Сборка и запуск из коробки**
   - Приложение собирается и запускается из коробки после клонирования репозитория и выполнения шагов из `docs/practices/00-getting-started.md` (Flutter, `flutter pub get`, эмулятор/симулятор). Без подстановки ключей доступны учёт расходов, обменник, портфель и все практики по трафику и инструментам (Charles, Proxyman, Android Studio, Xcode, ADB).

2. **Интеграции метрик и Firebase — только подстановка токенов**
   - Интеграции Sentry, AppMetrica и Firebase не требуют правок кода: достаточно подставить ключи в указанные места. **Sentry и AppMetrica** — единственное место: `lib/config/student_env.dart` (константы `sentryDsn`, `appMetricaApiKey`); при пустых значениях приложение работает без этих сервисов. **Firebase** — по инструкции `docs/practices/00-firebase-setup.md`: добавление своих конфигов `google-services.json` и `GoogleService-Info.plist` в проект.

3. **Функциональность**
   - Учёт расходов/доходов, обменник и мини-портфель работают на Android и iOS (и при необходимости на web).

4. **Трафик и снифф**
   - Запросы к API курсов идут по HTTPS; трафик виден в Charles и Proxyman при корректной настройке прокси на устройстве/эмуляторе.

5. **Практики**
   - В репозитории есть `docs/practices` с практиками по: Charles, Proxyman, Android Studio, Xcode, ADB, Sentry, AppMetrica, TestFlight, Android-дистрибуции, Firebase (00-firebase-setup, Crashlytics, FCM, Analytics, Remote Config, Performance, In-App Messaging).

6. **README**
   - Описание учебной цели продукта, ссылки на практики, требования для запуска (Flutter, эмуляторы/симуляторы).
