# Практика 14: Firebase Performance

## Технология

**Firebase Performance Monitoring** — замер времени загрузки экранов, сетевых запросов, кастомных трейсов. Тестировщику нужно уметь включать трейсы в коде и анализировать отчёты в консоли.

## В проекте FinControl

- После 00-firebase-setup подключается `firebase_performance`. Пример: трейс загрузки курсов (от начала запроса до отображения на экране Обменник), трейс открытия экрана.

## Задание

1. Подключить Firebase Performance в проекте.
2. Добавить кастомный трейс (например загрузка курсов) или использовать автоматические метрики.
3. Выполнить сценарий (открыть Обменник, дождаться загрузки курсов).
4. Проверить появление метрик/трейсов в Firebase Console → Performance.

**Полная инструкция:** [docs/practices/14-firebase-performance.md](../../practices/14-firebase-performance.md).
