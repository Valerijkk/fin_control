# 🤖 Практика 03: Android Studio — эмулятор, Logcat и инструменты тестирования

> Создаём эмулятор Android (AVD), устанавливаем FinControl, настраиваем **Logcat** для просмотра логов в реальном времени и осваиваем встроенные инструменты Android Studio для тестирования: Layout Inspector, Database Inspector, Profiler и другие.

---

## 🎯 Цель

Установить приложение FinControl в эмулятор Android Studio (AVD), запустить его и настроить просмотр логов через **Logcat** по процессу приложения — чтобы при работе с экранами (Список, Обменник, Портфель) видеть логи в реальном времени. Познакомиться с инструментами Android Studio для инспекции UI, базы данных, памяти и сети.

---

## ✅ Ожидаемый результат

- ✔ Создан и запущен эмулятор (AVD); приложение FinControl установлено (через `flutter run` или APK) и открывается с главного экрана
- ✔ В Android Studio открыта панель **Logcat**; выбраны устройство и процесс приложения (пакет из `android/app/build.gradle.kts`, например `com.yourname.fincontrol.fin_control`)
- ✔ При переходах по экранам и действиях в приложении в Logcat появляются новые строки логов (Flutter/Dart и системные)
- ✔ Освоены основные инструменты: Layout Inspector, App Inspection, Profiler

## 📝 Пошаговая инструкция

### Шаг 1: Создание и запуск эмулятора (AVD)

1. Открой **Android Studio** (проект FinControl должен быть открыт из корневой папки).
2. В меню выбери **Tools → Device Manager** (или **AVD Manager** в старых версиях).
3. Нажми **Create Device** → выбери устройство (например **Pixel 6**) → **Next**.
4. Выбери системный образ с **API 30 или выше** (при необходимости нажми **Download** рядом с образом) → **Next** → **Finish**.
5. В списке устройств нажми кнопку **Run** (▶) напротив созданного AVD.
6. Дождись загрузки эмулятора — появится окно с виртуальным устройством.

> 💡 Рекомендуем API 33–34 — это актуальные версии Android 13–14. Для тестирования совместимости можно создать несколько AVD с разными API.

---

### Шаг 2: Установка FinControl в эмулятор

#### Вариант A: из Flutter-проекта (рекомендуется)

Из корня проекта FinControl в терминале:

```bash
# Убедись, что эмулятор виден Flutter
flutter devices

# Запусти приложение — оно соберётся и установится автоматически
flutter run
```

Выбери запущенный эмулятор из списка, если устройств несколько.

#### Вариант B: установка готового APK

1. Собери APK:
   ```bash
   flutter build apk --debug
   ```
2. APK будет в `build/app/outputs/flutter-apk/app-debug.apk`.
3. Установи одним из способов:
   - **Перетащи** APK в окно эмулятора (drag & drop), или
   - Выполни команду:
     ```bash
     adb install -r build/app/outputs/flutter-apk/app-debug.apk
     ```
4. Запусти приложение на эмуляторе из лаунчера.

> ⚠️ Флаг `-r` означает переустановку с сохранением данных. Без него повторная установка вернёт ошибку.

---

### Шаг 3: Настройка Logcat в Android Studio

1. Открой панель **Logcat**: меню **View → Tool Windows → Logcat** или вкладка **Logcat** в нижней части окна.
2. В выпадающем списке **устройств** выбери свой запущенный эмулятор (например `emulator-5554`).
3. В выпадающем списке **процессов** выбери пакет приложения FinControl: `com.yourname.fincontrol.fin_control`.
   > 📌 Точное значение смотри в `android/app/build.gradle.kts` — строка `applicationId`. Так в логах будут видны только сообщения от твоего приложения.
4. Уровень логирования при необходимости измени на **Verbose**, **Debug** или **Info**.
5. Если в коде используются теги (например `FinControl`), в поле поиска Logcat введи этот тег для фильтрации.

---

### Шаг 4: Просмотр логов во время работы приложения

1. Запусти FinControl в эмуляторе (если ещё не запущен).
2. Выполни типичные сценарии:
   - Открой экран **Обменник** (сетевой запрос к API курсов)
   - Перейди на **Портфель**
   - Добавь расход на экране **Список**
3. В Logcat должны появляться сообщения от Flutter/Dart и нативные логи.

#### Продвинутые фильтры Logcat

| Фильтр | Что показывает |
|--------|----------------|
| `tag:FinControl` | Только логи с тегом `FinControl` |
| `tag:flutter` | Логи Flutter-движка |
| `level:error` | Только ошибки и крэши |
| `package:com.yourname.fincontrol.fin_control level:error` | Ошибки только от приложения |
| `Exception\|Error\|FATAL` (regex) | Все исключения и фатальные ошибки |

#### Сохранение логов в файл

В Logcat нажми **Export to text file** (иконка дискеты) или через ADB:

```bash
# Сохранить текущий буфер логов в файл с таймстампом
adb logcat -d > logcat_$(date +%Y%m%d_%H%M%S).txt
```

> 💡 Приложи файл логов к баг-репорту — это стандартная практика в тестировании.

## 🛠 Траблшутинг

**Эмулятор не появляется в списке при `flutter run`**
→ Сначала запусти эмулятор из Android Studio (**Tools → Device Manager → Run**), затем выполни `flutter devices` и `flutter run`.
→ [FAQ — Эмулятор не появляется в списке устройств](../FAQ.md#эмулятор-не-появляется-в-списке-устройств-при-flutter-run)

**В списке процессов нет пакета приложения**
→ Убедись, что приложение хотя бы раз было запущено на этом эмуляторе. Проверь package name в `android/app/build.gradle.kts`.
→ [FAQ — Где указан package name](../FAQ.md#где-в-проекте-указан-package-name-applicationid-приложения)

**Сборка завершается с ошибкой Gradle**
→ Выполни `flutter clean && flutter pub get` из корня проекта, затем попробуй снова.

**Эмулятор запускается, но экран чёрный**
→ Попробуй создать AVD заново с другим образом. Убедись, что включена аппаратная виртуализация (Intel HAXM или Hyper-V).

## 🧪 Практические сценарии

### Сценарий 1: Отслеживание логов при навигации

1. Запусти FinControl на эмуляторе.
2. Открой **Logcat**, в фильтре введи `[FinControl]`.
3. В приложении нажми **Начать** на приветственном экране.
4. **В Logcat увидишь:**
   ```
   [FinControl] Запуск приложения...
   [FinControl] Данные загружены: N записей, M категорий
   [FinControl] HomeScreen: initState — загрузка курсов и списка расходов
   ```
5. Перейди на **Обменник** → в Logcat:
   ```
   [FinControl] ExchangeScreen: курсы загружены (exchangerate.host), 10 валют
   ```
6. Сделай обмен → в Logcat:
   ```
   [FinControl] ExchangeScreen: обмен 1000,00 RUB → 10,50 USD (курс 0,01)
   ```

---

### Сценарий 2: Инспекция базы данных через App Inspection

1. **View → Tool Windows → App Inspection**.
2. Выбери процесс FinControl, вкладка **Database Inspector**.
3. Открой таблицу `expenses` — увидишь все расходы/доходы.
4. Добавь расход в приложении (Кофе, 350 руб., Еда).
5. В Database Inspector нажми **Refresh** — новая запись появится.
6. Выполни SQL-запрос:
   ```sql
   SELECT title, amount, category FROM expenses WHERE category = 'Еда';
   ```
7. **Что проверить:** запись «Кофе» присутствует, сумма 350, категория «Еда».

> 💡 Все таблицы SQLite приложения: `expenses`, `exchange_operations`, `portfolio_holdings`, `portfolio_transactions`, `price_alerts`, `limit_orders`, `savings_goals`. Можно выполнять любые SQL-запросы в реальном времени.

---

### Сценарий 3: Проверка утечек памяти через Memory Profiler

1. Открой **Profiler** (**View → Tool Windows → Profiler**) → выбери процесс FinControl → вкладка **Memory**.
2. Запомни текущий уровень памяти (например 80 MB).
3. В приложении: Список → Обменник → Портфель → Акции → назад → Обменник → назад (повтори 10 раз).
4. В Profiler нажми **Force Garbage Collection** (иконка мусорного бака).
5. **Проверь:** вернулась ли память к исходному уровню (~80 MB)?
6. Если память растёт после GC — возможна утечка. Нажми **Dump Java Heap** для анализа.

---

### Сценарий 4: Анализ сетевого трафика через Network Profiler

1. В **Profiler** выбери вкладку **Network**.
2. В приложении открой **Обменник** → нажми **Обновить**.
3. На графике появится всплеск — это запрос к API курсов.
4. Кликни на запрос → увидишь:
   - **URL:** `https://api.exchangerate.host/latest?base=RUB&symbols=...`
   - **Время выполнения** (в мс)
   - **Размер ответа** (в байтах)
   - **Статус:** 200
5. **Сравни** с Charles/Proxyman — данные должны совпадать.

> 💡 **Отличие от Charles/Proxyman:** Network Profiler показывает трафик без необходимости настройки прокси и сертификатов, но менее удобен для модификации запросов.

## 🔧 Дополнительные инструменты Android Studio

### Layout Inspector — проверка UI в реальном времени

Layout Inspector — визуальный инструмент для инспекции иерархии виджетов работающего приложения.

1. Запусти приложение на эмуляторе.
2. **Tools → Layout Inspector** → выбери процесс FinControl.
3. Откроется окно с 3D-представлением иерархии виджетов.
4. Кликни на элемент — в панели справа увидишь его свойства (размеры, отступы, текст, цвет).

**Что проверять:**
- Правильность вложенности элементов (нет ли лишних контейнеров)
- Размеры и отступы (соответствуют ли дизайну)
- Видимость элементов (не перекрывают ли друг друга)
- Текст (обрезается ли длинный текст, корректна ли локализация)

---

### CPU Profiler — производительность

1. В **Profiler** выбери **CPU**.
2. Нажми **Record** → выполни действие в приложении (обмен валюты) → **Stop**.
3. Увидишь flame chart — какие методы заняли больше всего времени.
4. Ищи: длинные операции на main thread (могут вызвать зависание UI).

---

### Device File Explorer — файловая система устройства

1. **View → Tool Windows → Device File Explorer**.
2. Навигация по файловой системе устройства/эмулятора.
3. Путь к данным приложения: `/data/data/com.yourname.fincontrol.fin_control/` (доступно в debug-сборке).
4. Здесь можно найти:
   - `databases/fin_control.db` — база данных SQLite
   - `shared_prefs/` — SharedPreferences
   - `cache/` — кэш приложения
5. Правый клик → **Save As** — скачай файл на ПК для анализа.
