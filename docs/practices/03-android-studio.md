# Практика: Android Studio — установка FinControl в эмуляторе и логирование

## Цель

Установить приложение FinControl в эмуляторе Android Studio и настроить просмотр логов (logcat) для отладки и тестирования.

## Что понадобится

- Android Studio с установленным Android SDK
- Проект FinControl (Flutter) или собранный APK

## Шаг 1: Создание и запуск эмулятора (AVD)

1. Откройте Android Studio → **Tools → Device Manager** (или **AVD Manager**).
2. **Create Device** → выберите устройство (например Pixel 6) → **Next**.
3. Выберите системный образ (рекомендуется API 30+), скачайте при необходимости → **Next** → **Finish**.
4. Запустите эмулятор кнопкой **Run** (▶) напротив созданного AVD.

## Шаг 2: Установка FinControl в эмулятор

### Вариант A: из Flutter-проекта

Из корня проекта FinControl в терминале:

```bash
flutter run
```

Выберите запущенный эмулятор из списка. Приложение соберётся и установится.

### Вариант B: установка готового APK

1. Соберите APK: `flutter build apk --debug` (или `--release`).
2. APK будет в `build/app/outputs/flutter-apk/app-debug.apk` (или `app-release.apk`).
3. Перетащите APK в окно эмулятора или выполните:
   ```bash
   adb install -r build/app/outputs/flutter-apk/app-debug.apk
   ```
4. Запустите приложение на эмуляторе из лаунчера.

## Шаг 3: Настройка logcat в Android Studio

1. **View → Tool Windows → Logcat** (или вкладка **Logcat** внизу).
2. В выпадающем списке устройств выберите ваш эмулятор.
3. В фильтре по процессу выберите пакет приложения: `com.yourname.fincontrol.fin_control` (или как указано в `android/app/build.gradle.kts` / `AndroidManifest.xml`).
4. Уровень логирования: **Verbose** / **Debug** / **Info** — по необходимости.
5. При использовании приложением тегов (например `FinControl`) введите в поиск Logcat этот тег.

## Шаг 4: Просмотр логов во время работы приложения

1. Запустите FinControl в эмуляторе.
2. Выполните сценарии: открытие экрана Обменник (сетевой запрос), Портфель, добавление расхода.
3. В Logcat должны появляться сообщения от Flutter/Dart и нативные логи (если они добавлены в коде).

## Что проверить (чек-лист)

- [ ] Эмулятор создан и запущен.
- [ ] FinControl установлен и открывается.
- [ ] В Logcat выбран процесс приложения, видны логи при действиях в приложении.

## Что показать на экзамене / созвоне

1. Показать запущенный эмулятор с открытым FinControl.
2. Открыть **Logcat** в Android Studio, выбрать устройство и процесс приложения (пакет `com.yourname.fincontrol.fin_control`).
3. В приложении перейти по экранам (Список → Обменник → Портфель) — в Logcat должны появляться новые строки (логи Flutter/системы).
4. Кратко сказать: «Создал AVD, установил приложение через Flutter run / APK. В Logcat отфильтровал по пакету приложения — при работе с экранами вижу логи.»

## Дополнительно

- Фильтр по уровню: **Error**, **Warn** — для поиска ошибок.
- Сохранение логов в файл: в Logcat меню **Export** или через `adb logcat` в терминале.
