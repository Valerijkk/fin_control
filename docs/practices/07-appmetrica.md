# 📊 Практика 07: AppMetrica — аналитика приложения FinControl

> Подключаем Яндекс AppMetrica к FinControl, отслеживаем сессии, устройства и кастомные события (обмен валюты, покупка акций, добавление расходов). Результат — в кабинете AppMetrica видны данные об использовании приложения и все пользовательские действия.

---

## 🎯 Цель

Зарегистрировать приложение FinControl в кабинете AppMetrica (Яндекс), получить **API Key**, вставить его в единственное место — `lib/config/student_env.dart` — и убедиться, что сессии, устройства и кастомные события появляются в отчётах AppMetrica.

---

## ✅ Ожидаемый результат

- ✔️ В кабинете [appmetrica.io](https://appmetrica.io) создано приложение FinControl; **API Key** скопирован
- ✔️ В файле **`lib/config/student_env.dart`** константа `appMetricaApiKey` заполнена твоим ключом; приложение перезапущено
- ✔️ После запуска приложения и навигации по экранам (1–2 минуты) в кабинете AppMetrica в разделах **Отчёты / Онлайн** появляются данные об установках, сессиях и устройствах (с задержкой 5–20 минут)
- ✔️ В разделе **События** видны кастомные события (`exchange_completed`, `portfolio_buy`, `expense_added` и др.)

> 📌 **Куда вставлять API Key:** только в [**lib/config/student_env.dart**](../../lib/config/student_env.dart) — константа `appMetricaApiKey`. Сохрани файл и перезапусти приложение — больше ничего не нужно. Подробнее: [STUDENT_ENV.md](../STUDENT_ENV.md).

## 📝 Пошаговая инструкция

### Шаг 1: Регистрация приложения в AppMetrica

1. Открой [appmetrica.io](https://appmetrica.io) и войди через Яндекс-аккаунт (или партнёрский вход).
2. Нажми **Добавить приложение**.
3. Укажи название — например **FinControl** — и выбери платформу **Android** и/или **iOS**.
4. После создания открой карточку приложения и скопируй **API Key** (числовой ключ). Он понадобится для вставки в `student_env.dart`.

> ⚠️ **Безопасность:** не коммить API Key в репозиторий. Вставляй его только в локальный `student_env.dart`; при push оставляй константу пустой. Подробнее: [STUDENT_ENV.md](../STUDENT_ENV.md) (раздел «Безопасность ключей»).

---

### Шаг 2: Вставка API Key в приложение

1. Открой файл [**lib/config/student_env.dart**](../../lib/config/student_env.dart) в любом редакторе.
2. Найди константу **`appMetricaApiKey`** (в начале файла подписано: «ПРАКТИКА 07 — APPMETRICA»).
3. Вставь скопированный API Key **между кавычками**:
   ```dart
   // ПРАКТИКА 07 — APPMETRICA
   const String appMetricaApiKey = 'твой_ключ_из_кабинета';
   ```
4. Сохрани файл.
5. **Перезапусти приложение** — выполни `flutter run` заново. Инициализация AppMetrica происходит в `lib/main.dart` при старте при непустом ключе (пакет `appmetrica_plugin` уже подключён).

> 💡 Добавлять код инициализации вручную **не нужно** — в FinControl он уже есть. Не дублируй API Key в других файлах.

---

### Шаг 3: Проверка — сессии и события в отчётах

1. Запусти приложение после сохранения `student_env.dart`.
2. Походи по экранам **1–2 минуты**: Список → Обменник → Портфель → Статистика.
3. Выполни несколько действий: обменяй валюту, добавь расход, купи акцию — это отправит кастомные события.
4. В браузере открой кабинет AppMetrica → твоё приложение FinControl.
5. Перейди в раздел **Отчёты → Стандартные отчёты** или **Онлайн**.
6. Учитывай задержку: **5–20 минут** (при первом запуске может быть дольше). Обнови страницу или зайди позже.
7. В отчётах должны появиться: **Установки**, **Сессии**, **Устройства** (модель, ОС, версия приложения).

> 📌 Задержка 5–20 минут — это нормальное поведение AppMetrica. Не паникуй, если данные не появились сразу.

## 🧪 Практические сценарии

### Сценарий 1: Проверка сессий и устройств

1. Запусти приложение с заполненным `appMetricaApiKey`.
2. Походи по экранам 2–3 минуты: Список → Обменник → Портфель → Акции → Статистика.
3. Открой [appmetrica.io](https://appmetrica.io) → твоё приложение.
4. Подожди 5–20 минут → обнови отчёты.
5. **Что проверить:**
   - **Стандартные отчёты → Установки** — должна быть минимум 1 установка
   - **Сессии** — должна быть хотя бы 1 сессия
   - **Устройства** — модель эмулятора/телефона, версия ОС

---

### Сценарий 2: Отслеживание кастомных событий

FinControl автоматически отправляет следующие события в AppMetrica (если `appMetricaApiKey` заполнен):

| Событие | Когда срабатывает | Параметры |
|---------|-------------------|-----------|
| `rates_loaded` | При загрузке курсов на экране Обменник | — |
| `exchange_completed` | После обмена валюты | `currency_from`, `currency_to`, `amount` |
| `portfolio_buy` | Покупка валюты в портфеле | `currency`, `amount` |
| `portfolio_sell` | Продажа из портфеля | `currency`, `amount` |
| `expense_added` | Добавление расхода/дохода | `category`, `amount`, `is_income`, `source` |
| `stock_buy` | Покупка акций | `ticker`, `shares`, `cost_rub` |
| `crypto_buy` | Покупка криптовалюты | `symbol`, `amount`, `cost_rub` |
| `theme_toggled` | Переключение темы | `new_theme` |
| `data_cleared` | Очистка всех данных | — |

**Как проверить:**

1. Выполни обмен валюты в Обменнике.
2. Купи акцию или валюту в Портфеле.
3. Добавь расход через быструю запись.
4. В AppMetrica → **Отчёты → События** через 5–20 минут увидишь все события с параметрами.

---

### Сценарий 3: Построение воронки

1. Открой AppMetrica → **Воронки** (или Funnel).
2. Построй воронку:
   - **Шаг 1:** `rates_loaded` (пользователь открыл Обменник)
   - **Шаг 2:** `exchange_completed` (совершил обмен)
3. **Вопрос:** какой процент пользователей, загрузивших курсы, реально совершает обмен?
4. В учебном проекте это будет 100% (один пользователь) — но в реальных проектах это ключевая метрика конверсии.

---

### Сценарий 4: Сравнение источников добавления расхода

1. Добавь **3 расхода** через **быструю запись** (FAB с молнией).
2. Добавь **2 расхода** через **полную форму** (FAB с +).
3. В AppMetrica → **События → expense_added** → группируй по параметру `source`.
4. Увидишь: 3 события с `source: quick_add` и 2 с `source: full_form`.
5. **Зачем:** в реальном проекте это покажет, какой способ добавления предпочитают пользователи — ценная информация для продукт-менеджера.

## 📚 Дополнительно: углублённая работа с AppMetrica

### Отправка кастомных событий

AppMetrica позволяет отслеживать конкретные действия пользователя:

```dart
import 'package:appmetrica_plugin/appmetrica_plugin.dart';

// Простое событие
AppMetrica.reportEvent('exchange_completed');

// Событие с параметрами
AppMetrica.reportEventWithMap('portfolio_buy', {
  'currency': 'USD',
  'amount': '100',
  'rate': '92.5',
});

// Событие добавления расхода
AppMetrica.reportEventWithMap('expense_added', {
  'category': 'Еда',
  'amount': '500',
  'is_income': 'false',
});
```

В кабинете AppMetrica → **Отчёты → События** увидишь все события с параметрами.

### User Profile — профиль пользователя

```dart
// Идентификация пользователя (анонимная для учебного проекта)
AppMetrica.setUserProfileID('student-001');
```

### Экраны и сессии

AppMetrica автоматически отслеживает сессии. Для трекинга экранов:

```dart
// При открытии экрана
AppMetrica.reportEvent('screen_view_exchange');
AppMetrica.reportEvent('screen_view_portfolio');
```

---

### Отчёты в консоли AppMetrica

| Раздел | Что покажет |
|--------|------------|
| **Стандартные отчёты** | Установки, сессии, устройства, ОС, география |
| **События** | Список всех отправленных событий с параметрами |
| **Профили** | Информация об отдельных пользователях |
| **Когорты** | Удержание пользователей по дням |
| **Воронки** | Конверсия по цепочке событий (например `screen_view_exchange` → `exchange_completed`) |

---

### Revenue — отслеживание «покупок»

Для учебного сценария (покупка валюты в портфеле):

```dart
AppMetrica.reportEventWithMap('revenue', {
  'currency': 'RUB',
  'amount': '5000',
  'product': 'USD',
});
```

> 💡 В реальных проектах используется `AppMetrica.reportRevenue()` с SDK-методом для точного учёта покупок.
