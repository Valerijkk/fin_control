# Практика: AppMetrica — заведение приложения и проверка

**Одно приложение** FinControl. Всё из коробки: нужно только **вставить API Key в `lib/config/student_env.dart`** и перезапустить приложение. Проверка за **1–2 действия:** запусти приложение, походи по экранам 1–2 минуты → в кабинете AppMetrica (Отчёты / Онлайн) через 5–20 минут появятся сессии и устройства.

## Цель

Зарегистрировать одно приложение FinControl в кабинете AppMetrica (Яндекс), получить API Key, вставить его в единственное место в коде — `lib/config/student_env.dart` — и проверить, что сессии и устройства появляются в отчётах AppMetrica после запуска приложения и навигации по экранам.

## Ожидаемый результат

- В кабинете [appmetrica.io](https://appmetrica.io) создано приложение (Android и/или iOS) с именем FinControl; скопирован **API Key**.
- В файле **`lib/config/student_env.dart`** в константу `appMetricaApiKey` вставлен ваш API Key; приложение перезапущено.
- После запуска приложения и действий (Список → Обменник → Портфель) в кабинете AppMetrica в разделах **Отчёты** / **Онлайн** (с задержкой 5–20 минут) появляются данные об установках, сессиях и устройствах.

**Куда вставлять API Key:** только в [**lib/config/student_env.dart**](../../lib/config/student_env.dart) — константа `appMetricaApiKey`. Сохрани файл и перезапусти приложение — больше ничего не нужно. Подробнее: [STUDENT_ENV.md](../STUDENT_ENV.md).

## Что понадобится

- Аккаунт в [AppMetrica](https://appmetrica.io) (Яндекс)
- Проект FinControl с подключённым AppMetrica SDK для Flutter

## Шаг 1: Регистрация приложения в AppMetrica

1. Перейди на [appmetrica.io](https://appmetrica.io) и войди (через Яндекс или партнёрский вход).
2. Создай приложение: **Добавить приложение** → укажи название (например FinControl), платформу **iOS** и/или **Android**.
3. После создания открой карточку приложения и скопируй **API Key** (числовой ключ). Он понадобится для вставки в `student_env.dart`.

**Безопасность:** не коммитьте API Key и конфиги с ключами в репозиторий. Вставляйте API Key только в локальный `student_env.dart`; при push оставляйте константу пустой. См. [STUDENT_ENV.md](../STUDENT_ENV.md) (раздел «Безопасность ключей»).

## Шаг 2: Куда вставить API Key в приложении

1. В проекте FinControl API Key задаётся **в одном файле**. Открой [**lib/config/student_env.dart**](../../lib/config/student_env.dart).
2. Найди константу **`appMetricaApiKey`** (в начале файла капсом подписано: «ПРАКТИКА 07 — APPMETRICA»). Вставь скопированный API Key **в пустую строку между кавычками** (сейчас там `''`). После подстановки будет, например: `const String appMetricaApiKey = 'твой_ключ_из_кабинета';` — не коммить в репозиторий заполненное значение, при push оставляй `''`.
3. Сохрани файл. **Перезапусти приложение** (`flutter run` заново) — инициализация AppMetrica выполняется в `lib/main.dart` при старте при непустом ключе (пакет `appmetrica_plugin` уже подключён в FinControl).
4. Не дублируй API Key в других файлах; добавлять код инициализации вручную не нужно.

## Шаг 3: Проверка за 1–2 действия

1. Запусти приложение после сохранения `student_env.dart` и походи по экранам: Список, Обменник, Портфель (1–2 минуты).
2. В браузере открой кабинет AppMetrica → твоё приложение FinControl.
3. Перейди в раздел **Отчёты** / **Стандартные отчёты** или **Онлайн**.
4. Учитывай задержку обработки данных: **5–20 минут** (при первом запуске может быть дольше). Обнови страницу или зайди позже.
5. В отчётах должны появиться **Установки**, **Сессии**, **Устройства** (модель, ОС, версия приложения).

## Практические сценарии работы с AppMetrica в FinControl

### Сценарий 1: Проверка сессий и устройств

1. Запусти приложение с заполненным `appMetricaApiKey`.
2. Походи по экранам 2-3 минуты: Список → Обменник → Портфель → Акции → Статистика.
3. Открой [appmetrica.io](https://appmetrica.io) → твоё приложение.
4. Подожди 5-20 минут → обнови отчёты.
5. **Что проверить:**
   - **Стандартные отчёты → Установки** — должна быть 1 установка.
   - **Сессии** — должна быть хотя бы 1 сессия.
   - **Устройства** — модель эмулятора/телефона, версия ОС.

### Сценарий 2: Отслеживание кастомных событий

FinControl отправляет следующие события в AppMetrica (если `appMetricaApiKey` заполнен):

| Событие | Когда | Параметры |
|---------|-------|-----------|
| `rates_loaded` | При загрузке курсов на экране Обменник | — |
| `exchange_completed` | После обмена валюты | `currency_from`, `currency_to`, `amount` |
| `portfolio_buy` | Покупка валюты в портфеле | `currency`, `amount` |
| `portfolio_sell` | Продажа из портфеля | `currency`, `amount` |
| `expense_added` | Добавление расхода/дохода | `category`, `amount`, `is_income`, `source` |
| `stock_buy` | Покупка акций | `ticker`, `shares`, `cost_rub` |
| `crypto_buy` | Покупка криптовалюты | `symbol`, `amount`, `cost_rub` |
| `theme_toggled` | Переключение темы | `new_theme` |
| `data_cleared` | Очистка всех данных | — |

**Как проверить:**
1. Выполни обмен валюты в Обменнике.
2. Купи акцию или валюту в Портфеле.
3. Добавь расход через быструю запись.
4. В AppMetrica → **Отчёты → События** через 5-20 минут увидишь все события с параметрами.

### Сценарий 3: Построение воронки

1. Открой AppMetrica → **Воронки** (или Funnel).
2. Построй воронку:
   - Шаг 1: `rates_loaded` (пользователь открыл Обменник)
   - Шаг 2: `exchange_completed` (совершил обмен)
3. **Вопрос:** какой процент пользователей, загрузивших курсы, реально совершает обмен?
4. В учебном проекте это будет 100% (один пользователь) — но в реальных проектах это ключевая метрика.

### Сценарий 4: Сравнение источников добавления расхода

1. Добавь 3 расхода через **быструю запись** (FAB с молнией).
2. Добавь 2 расхода через **полную форму** (FAB с +).
3. В AppMetrica → **События → expense_added** → группируй по параметру `source`.
4. Увидишь: 3 события с `source: quick_add` и 2 с `source: full_form`.
5. **Зачем:** в реальном проекте это покажет, какой способ добавления предпочитают пользователи.

## Дополнительно: углублённая работа с AppMetrica

### Отправка кастомных событий

AppMetrica позволяет отслеживать конкретные действия пользователя:

```dart
import 'package:appmetrica_plugin/appmetrica_plugin.dart';

// Событие обмена валюты
AppMetrica.reportEvent('exchange_completed');

// Событие с параметрами
AppMetrica.reportEventWithMap('portfolio_buy', {
  'currency': 'USD',
  'amount': '100',
  'rate': '92.5',
});

// Событие добавления расхода
AppMetrica.reportEventWithMap('expense_added', {
  'category': 'Еда',
  'amount': '500',
  'is_income': 'false',
});
```

В кабинете AppMetrica → **Отчёты → События** увидишь все события с параметрами.

### User Profile — профиль пользователя

```dart
// Идентификация пользователя (анонимная для учебного проекта)
AppMetrica.setUserProfileID('student-001');
```

### Экраны и сессии

AppMetrica автоматически отслеживает сессии. Для трекинга экранов:

```dart
// При открытии экрана
AppMetrica.reportEvent('screen_view_exchange');
AppMetrica.reportEvent('screen_view_portfolio');
```

### Отчёты в консоли

В кабинете AppMetrica доступны:
- **Стандартные отчёты:** установки, сессии, устройства, ОС, география.
- **События:** список всех отправленных событий с параметрами.
- **Профили:** информация об отдельных пользователях.
- **Когорты:** удержание пользователей по дням.
- **Воронки:** можно построить по событиям (например screen_view_exchange → exchange_completed).

### Revenue — отслеживание «покупок»

Для учебного сценария (покупка валюты в портфеле):

```dart
AppMetrica.reportEventWithMap('revenue', {
  'currency': 'RUB',
  'amount': '5000',
  'product': 'USD',
});
```

В реальных проектах используется `AppMetrica.reportRevenue()` с SDK-методом для точного учёта.
