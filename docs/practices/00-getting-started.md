# Перед началом любых практик

Клонирование репозитория, первый запуск, открытие проекта в Android Studio. Выполни эти шаги до практик 01–15.

---

## 1. Клонирование и первый запуск

**Что понадобится:** установленные Git и Flutter (проверка — команды `git --version` и `flutter --version` в терминале).

### Шаг 1. Открой терминал

- **Windows:** PowerShell (Win + X → Windows PowerShell) или cmd. Можно также встроенный терминал в Cursor/VS Code (Ctrl+`).
- **Mac/Linux:** Terminal или встроенный терминал в IDE.

### Шаг 2. Перейди в папку для проектов

- Пример: `cd Desktop` или `cd C:\Users\ИМЯ\Desktop`, или `cd ~/Projects`.
- Убедись, что ты не внутри другого репозитория (не должно быть скрытой папки `.git` в текущей директории, если хочешь положить fin_control рядом с другими проектами).

### Шаг 3. Клонируй репозиторий

```bash
git clone https://github.com/Valerijkk/fin_control.git
cd fin_control
```

- Если `git clone` выдаёт ошибку «command not found» — установи Git с [git-scm.com](https://git-scm.com/).
- Если ошибка доступа/таймаут — проверь интернет и при необходимости настрой прокси для Git.

### Шаг 4. Проверь Flutter и установи зависимости

```bash
flutter doctor
```

- Смотри вывод: для практик достаточно, чтобы был отмечен галочкой **Android** (или iOS на Mac). Красные крестики по Android Toolchain или Flutter нужно устранить (установить SDK, принять лицензии: `flutter doctor --android-licenses`).

```bash
flutter pub get
```

- Должно завершиться без ошибок. Это подтягивает пакеты из `pubspec.yaml`.

### Шаг 5. Запусти приложение

```bash
flutter run
```

- Если устройств несколько — в терминале появится нумерованный список (например `[1] Pixel 9 API 34`, `[2] Chrome`). Введи номер нужного и Enter. Эмулятор нужно заранее запустить из Android Studio (Tools → Device Manager → Run).
- Первый запуск может занять 1–3 минуты (сборка Gradle, компиляция).
- **Что должно получиться:** на экране приложения приветственный экран с заголовком «FinControl» и кнопкой «Начать». После нажатия «Начать» открывается главный экран с нижней навигацией: **Список**, **Обменник**, **Акции**, **Портфель**, **Статистика**.

**Если не запускается:** типичные причины и решения — в разделе «Запуск и сборка» в [FAQ](../FAQ.md). Кратко: проверь `flutter doctor`, что открыт корень проекта (не папка `android/`), выполни `flutter clean` и снова `flutter pub get`.

## 2. Как открывать проект в Android Studio (обязательно)

**Почему это важно:** в Android Studio нужно открывать **корневую папку проекта** — ту, где лежат файл `pubspec.yaml` и папки `lib/`, `android/`, `ios/`, `docs/`. **Не** открывай папку `android/` саму по себе: тогда IDE считает проект обычным Android-приложением, не видит Flutter/Dart, и появляются сотни ошибок Gradle и красные подчёркивания по всему коду.

### Пошагово

1. Запусти Android Studio. Если уже открыт другой проект — **File → Close Project** (вернёшься на экран приветствия).
2. На экране приветствия нажми **Open** (или **File → Open**).
3. В диалоге выбора папки укажи **папку fin_control** — именно ту, внутри которой лежат `pubspec.yaml`, `lib`, `android`, `ios`. Не выбирай папку `android` внутри неё.
4. Нажми **OK**. Дождись окончания индексации (внизу статус «Indexing…») и при необходимости **Gradle sync** (может предложить сам или через **File → Sync Project with Gradle Files**).
5. **File → Settings** (на Mac: **Android Studio → Settings**) → **Plugins**. Убедись, что установлены и включены **Flutter** и **Dart**. Если их нет — вкладка **Marketplace**, поиск «Flutter», установка, затем перезапуск Android Studio.

**Как проверить, что открыто правильно:** в дереве проекта (Project) должны быть видны папки `android`, `ios`, `lib`, `test`, файл `pubspec.yaml` в корне. При открытии папки `lib` файлы должны подсвечиваться как Dart (подсветка синтаксиса).

Подробнее и решение проблем: [ANDROID_STUDIO_LAUNCH.md](../technical/ANDROID_STUDIO_LAUNCH.md) (в технической документации).
