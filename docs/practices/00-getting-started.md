# 🚀 Перед началом: клонирование, первый запуск и Android Studio

> **Что делаем:** клонируем репозиторий FinControl, устанавливаем зависимости, запускаем приложение на эмуляторе и правильно открываем проект в Android Studio.
> **Зачем:** это фундамент — без этих шагов ни одна практика (01–15) не заработает.
> **Результат:** приложение FinControl запущено, Android Studio видит Flutter-проект.

---

## 🎯 Цель

Подготовить рабочее окружение: склонировать репозиторий FinControl, установить зависимости, запустить приложение на устройстве или эмуляторе и открыть проект в Android Studio так, чтобы IDE корректно распознавала Flutter-проект.

> 📌 **Одно приложение** — всё из коробки: после этих шагов FinControl уже работает без вставки ключей. Ключи Sentry/AppMetrica понадобятся только в практиках 06–07 ([STUDENT_ENV.md](../STUDENT_ENV.md)), Firebase — в практике 10 ([10-firebase.md](10-firebase.md)).

## ✅ Ожидаемый результат

- ✔️ Репозиторий лежит на диске, из корня проекта выполняются `flutter pub get` и `flutter run`
- ✔️ Приложение **запускается из коробки**: приветственный экран «FinControl» → кнопка «Начать» → главный экран с навигацией (Список, Обменник, Акции, Портфель, Статистика). Никаких токенов для старта не нужно
- ✔️ В Android Studio открыта **корневая** папка проекта; в дереве видны `lib/`, `android/`, `ios/`, `pubspec.yaml`; нет массовых ошибок Gradle
- ✔️ Подсветка синтаксиса Dart работает, плагины Flutter и Dart установлены

## 📝 Пошаговая инструкция

### Шаг 1: Открой терминал

1. **Windows:** PowerShell (Win + X → Windows PowerShell) или cmd. Можно встроенный терминал в VS Code (`Ctrl + ~`) или Android Studio (`Alt + F12`)
2. **Mac/Linux:** Terminal или встроенный терминал в IDE

> 💡 Весь дальнейший ввод команд — в этом терминале. Не закрывай его до конца инструкции.

---

### Шаг 2: Перейди в папку для проектов

1. Выбери место, где будет лежать проект. Например:
   ```bash
   # Windows
   cd C:\Users\ИМЯ\Desktop

   # Mac / Linux
   cd ~/Desktop
   ```
2. Убедись, что ты **не внутри другого репозитория** — в текущей папке не должно быть скрытой папки `.git`

> ⚠️ Не клонируй проект внутрь другого Git-репозитория — это вызовет конфликты.

---

### Шаг 3: Клонируй репозиторий

1. Выполни команду:
   ```bash
   git clone https://github.com/Valerijkk/fin_control.git
   ```
2. Перейди в папку проекта:
   ```bash
   cd fin_control
   ```
3. Убедись, что ты внутри проекта — должны быть видны файлы:
   ```bash
   ls
   # Ожидаемый вывод: android/  ios/  lib/  docs/  pubspec.yaml  ...
   ```

> ⚠️ Если `git clone` выдаёт ошибку **«command not found»** — установи Git с [git-scm.com](https://git-scm.com/). Если ошибка доступа или таймаут — проверь интернет-соединение и при необходимости настрой прокси для Git.

---

### Шаг 4: Проверь Flutter и установи зависимости

1. Запусти диагностику Flutter:
   ```bash
   flutter doctor
   ```
   - Для практик достаточно, чтобы **Android toolchain** был отмечен ✅ (или iOS на Mac)
   - Если есть красные ❌ по Android — установи Android SDK и прими лицензии:
     ```bash
     flutter doctor --android-licenses
     ```

2. Установи зависимости проекта:
   ```bash
   flutter pub get
   ```
   - Команда должна завершиться без ошибок — это подтягивает все пакеты из `pubspec.yaml`

> 💡 Если `flutter pub get` падает с ошибкой — убедись, что ты находишься в **корне проекта** (там, где лежит `pubspec.yaml`), а не в подпапке `android/` или `lib/`.

---

### Шаг 5: Запусти приложение

1. Убедись, что эмулятор запущен (Android Studio → Tools → Device Manager → ▶️ напротив устройства) или подключено физическое устройство с включённой USB-отладкой
2. Выполни:
   ```bash
   flutter run
   ```
3. Если устройств несколько — в терминале появится нумерованный список (например `[1] Pixel 9 API 34`, `[2] Chrome`). Введи номер нужного и нажми Enter

> ⚠️ Первый запуск может занять **1–3 минуты** — идёт сборка Gradle и компиляция. Это нормально, не прерывай процесс.

4. **Что должно появиться на экране:**
   - Приветственный экран с заголовком **«FinControl»** и кнопкой **«Начать»**
   - После нажатия «Начать» — главный экран с нижней навигацией: **Список**, **Обменник**, **Акции**, **Портфель**, **Статистика**

---

### Шаг 6: Открой проект в Android Studio (обязательно)

> ⚠️ **Критически важно:** в Android Studio нужно открывать **корневую папку проекта** — ту, где лежат `pubspec.yaml` и папки `lib/`, `android/`, `ios/`. **Не открывай папку `android/` отдельно** — иначе IDE воспримет проект как обычное Android-приложение, не увидит Flutter/Dart, и появятся сотни ошибок Gradle.

1. Запусти Android Studio. Если уже открыт другой проект — **File → Close Project** (вернёшься на экран приветствия)
2. На экране приветствия нажми **Open** (или **File → Open**)
3. В диалоге выбора папки укажи **папку `fin_control`** — именно ту, внутри которой лежат `pubspec.yaml`, `lib/`, `android/`, `ios/`. **Не выбирай** папку `android/` внутри неё
4. Нажми **OK**. Дождись окончания индексации (внизу статус-бар покажет «Indexing…») и при необходимости **Gradle sync** (может предложить автоматически или через **File → Sync Project with Gradle Files**)
5. Проверь плагины: **File → Settings** (на Mac: **Android Studio → Settings**) → **Plugins**. Убедись, что установлены и включены:
   - **Flutter**
   - **Dart**

   Если их нет — перейди на вкладку **Marketplace**, найди «Flutter», установи, затем перезапусти Android Studio

> 💡 После установки плагинов и перезапуска Android Studio заново проиндексирует проект — подсветка Dart заработает, ошибки исчезнут.

## 🛠 Траблшутинг

| Проблема | Решение |
|---------|---------|
| **`flutter` — command not found** | Flutter SDK не установлен или не добавлен в PATH. Установи с [flutter.dev](https://flutter.dev/docs/get-started/install) и перезапусти терминал |
| **`git clone` — command not found** | Установи Git с [git-scm.com](https://git-scm.com/) |
| **`flutter.sdk` not set / Gradle sync failed** | Открой проект из **корня** `fin_control`, а не из `android/`. Выполни `flutter clean && flutter pub get` |
| **Эмулятор не в списке устройств** | Запусти эмулятор из Android Studio → Tools → Device Manager → ▶️. Подожди 10–15 секунд, затем повтори `flutter run` |
| **«No pubspec.yaml found»** | Ты запускаешь команду не из корня проекта. Перейди в папку `fin_control` командой `cd fin_control` |
| **Сотни ошибок и красных подчёркиваний в IDE** | Ты открыл папку `android/` вместо корня проекта. Закрой проект (File → Close Project) и открой заново правильную папку |
| **Первая сборка зависает надолго** | Это нормально — Gradle скачивает зависимости. Убедись, что интернет работает, и подожди до 5 минут |
