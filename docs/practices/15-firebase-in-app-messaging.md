# Практика: Firebase In-App Messaging

**Одно приложение** FinControl. **Сначала [00-firebase-setup.md](00-firebase-setup.md):** свой проект Firebase, конфиги в проект, `Firebase.initializeApp()`. Желательно [12-firebase-analytics.md](12-firebase-analytics.md) — In-App Messaging триггерится событиями Analytics. Затем создаёшь кампанию в консоли и проверяешь показ сообщения в приложении.

## Цель

Подключить Firebase In-App Messaging к приложению FinControl (твой проект Firebase), создать тестовую кампанию в консоли (триггер по событию Analytics или по экрану) и проверить показ in-app сообщения в приложении при срабатывании триггера.

## Важно: один проект Firebase

**Сначала [00-firebase-setup.md](00-firebase-setup.md):** зарегистрируй свой проект, добавь свои конфиги и `Firebase.initializeApp()` в коде. Ключи Sentry/AppMetrica (практики 06–07) — в [STUDENT_ENV.md](../STUDENT_ENV.md), не здесь. In-App Messaging использует события Analytics — желательно выполнить [12-firebase-analytics.md](12-firebase-analytics.md) для логирования экранов/событий.

## Ожидаемый результат

- Кампания создана в Firebase Console → **Engage** → **In-App Messaging**: шаблон (модальное окно, баннер и т.д.), текст, триггер (событие или экран).
- При запуске приложения и выполнении действия, соответствующего триггеру (открытие экрана или событие), сообщение показывается поверх приложения; в консоли видны показы и клики.

## Что понадобится

- Ваш Firebase-проект с подключённым приложением
- Добавьте в `pubspec.yaml` при необходимости `firebase_in_app_messaging` (совместимую с firebase_core)

## Шаг 1: Подключение

Добавьте при необходимости `firebase_in_app_messaging` в `pubspec.yaml`, выполните `flutter pub get`. В `main.dart` должен быть вызван `Firebase.initializeApp()` (по [00-firebase-setup.md](00-firebase-setup.md)). После этого In-App Messaging активируется и показывает кампании, созданные в **вашей** консоли Firebase. Дополнительный код в приложении чаще всего не требуется.

## Шаг 2: Создание кампании в консоли

1. Firebase Console → **Engage** → **In-App Messaging** (или **Messaging** → **In-app**).
2. **Create campaign** → **In-app message**.
3. Выберите шаблон (модальное окно, баннер, карточка и т.д.), введите заголовок и текст (например «Добро пожаловать в FinControl! Проверьте обменник.»).
4. **Targeting**:
   - По событию: выберите событие Analytics (например `screen_view` с параметром экрана или кастомное `exchange_completed`).
   - По экрану: укажите имя экрана (если логируете через Analytics).
5. Установите даты показа и при необходимости лимит показов. **Publish** кампанию.

## Шаг 3: Проверка в приложении

1. Запустите приложение и выполните действие, соответствующее триггеру (откройте нужный экран или выполните событие).
2. Сообщение должно появиться поверх приложения в соответствии с выбранным шаблоном.
3. В консоли в отчётах кампании можно посмотреть показы и клики.

## Проверка

- [ ] Выполнен [00-firebase-setup.md](00-firebase-setup.md).
- [ ] Кампания создана в Firebase Console → **Engage** → **In-App Messaging** и опубликована (Publish).
- [ ] Триггер (событие Analytics или имя экрана) настроен и реально выполняется в приложении при ваших действиях.
- [ ] In-app сообщение отображается в приложении при срабатывании триггера.

## Что показать на экзамене / созвоне

1. Покажи Firebase Console → Engage → In-App Messaging → твою кампанию (опубликована).
2. Запусти приложение → выполни действие-триггер (открой нужный экран или вызови событие).
3. Покажи in-app сообщение поверх приложения.
4. В консоли покажи статистику кампании (показы, клики).
5. Кратко скажи: «Создал кампанию In-App Messaging, настроил триггер по событию Analytics. При открытии экрана сообщение показывается поверх приложения.»

## Траблшутинг

- **Сообщение не показывается** — убедись, что триггерное событие/экран отправляется в Analytics (выполни практику 12 или проверь DebugView); подожди несколько минут после публикации кампании; проверь, что приложение подключено к тому же Firebase-проекту (твои конфиги из [00-firebase-setup.md](00-firebase-setup.md)). [FAQ — Firebase](../FAQ.md#firebase).
- **Показывается только один раз** — по умолчанию может быть лимит «один раз на пользователя»; измени настройки кампании при необходимости.
- **Сообщение показывается с задержкой** — In-App Messaging может кэшировать конфиг; для тестирования используй Test Mode (см. ниже).

## Дополнительно: сценарии для FinControl

### Рекомендуемые кампании

| Кампания | Шаблон | Триггер | Текст |
|----------|--------|---------|-------|
| Приветствие | Modal | `screen_view` (Welcome) | «Добро пожаловать! Начните с добавления расхода.» |
| Обменник | Banner (Top) | `screen_view` (Exchange) | «Совет: следите за курсами и создавайте ценовые алерты!» |
| Первая покупка | Card | `portfolio_buy` | «Поздравляем с первой покупкой! Проверьте портфель.» |
| Достижение цели | Modal | `goal_progress_100` | «Цель достигнута! Поставьте новую цель в настройках.» |

### Test Mode — быстрая проверка

Для тестирования без ожидания:

1. В Firebase Console → In-App Messaging → твоя кампания → **Test on Device**.
2. Укажи **Installation ID** приложения (можно получить через `FirebaseInstallations.instance.getId()`).
3. Сообщение покажется при следующем запуске приложения — без задержки.

```dart
// Получить Installation ID для тестирования
import 'package:firebase_core/firebase_core.dart';
final id = await FirebaseInstallations.instance.getId();
debugPrint('Installation ID: $id');
```

### Шаблоны сообщений

- **Modal** — модальное окно с заголовком, текстом, изображением и кнопкой. Для важных сообщений.
- **Banner (Top/Bottom)** — неинвазивный баннер, не блокирует UI. Для подсказок.
- **Card** — карточка с изображением. Для промо и поздравлений.
- **Image Only** — только изображение. Для визуальных кампаний.

### Связь с Analytics

In-App Messaging работает в связке с Firebase Analytics:
- Триггеры — события Analytics (`screen_view`, кастомные события из практики 12).
- Показы и клики кампаний записываются как события Analytics — можно строить воронки.
- Условия показа можно привязать к аудиториям Analytics (User Properties).
