# Практика: Firebase Performance Monitoring — регресс по скорости

## Цель

Подключить Firebase Performance Monitoring к FinControl, собрать автоматические трейсы (запуск приложения, сетевые запросы) и научиться находить регрессии по скорости в консоли.

## Важно: свой проект Firebase

**Сначала [00-firebase-setup.md](00-firebase-setup.md):** свой проект, свои конфиги, `Firebase.initializeApp()`.

## Что понадобится

- Ваш Firebase-проект с включённым Performance (в консоли)
- Добавьте в `pubspec.yaml` при необходимости `firebase_performance` (совместимую с firebase_core)

## Шаг 1: Подключение

Добавьте при необходимости `firebase_performance` в `pubspec.yaml`, выполните `flutter pub get`. После `Firebase.initializeApp()` (по 00-firebase-setup.md) Performance обычно начинает автоматически собирать:

- **App start** — время до первого кадра.
- **HTTP-запросы** — если использовать перехват (см. документацию пакета для Flutter).

Для кастомных операций создайте трейс:

```dart
import 'package:firebase_performance/firebase_performance.dart';

final trace = FirebasePerformance.instance.newTrace('load_rates');
await trace.start();
try {
  await RatesApi.fetch();
} finally {
  await trace.stop();
}
```

Добавьте такой трейс вокруг загрузки курсов (в экране обменника или главном экране), чтобы видеть время загрузки курсов в отчётах.

## Шаг 2: Просмотр в Firebase Console

1. **Performance** → **Dashboard**: обзор по метрикам (время запуска, время до интерактивности и т.д.).
2. **Traces** — список кастомных и автоматических трейсов; можно сравнить по версиям приложения и устройствам.
3. **Network** — если настроен перехват HTTP, видны запросы к API курсов и их длительность.

## Шаг 3: Поиск регрессий

- Сравните метрики между версиями (например до и после изменения кода).
- Фильтруйте по устройству/ОС — регресс может проявляться только на части устройств.
- Используйте кастомный трейс `load_rates`, чтобы отслеживать регресс скорости загрузки курсов.

## Что проверить

- [ ] Performance Monitoring включён, данные появляются в консоли (с задержкой до 24 ч для части отчётов).
- [ ] Кастомный трейс (например загрузки курсов) добавлен и отображается в Traces.
- [ ] По отчётам можно оценить время ключевых операций и сравнить версии.
