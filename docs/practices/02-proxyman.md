# Практика: Proxyman — перехват трафика FinControl

## Цель

Настроить Proxyman и перехватывать HTTP/HTTPS-запросы приложения FinControl к API курсов валют (Android и при возможности iOS).

## Что понадобится

- Proxyman ([proxyman.io](https://proxyman.io/)) — для macOS или Windows в зависимости от вашей ОС
- Устройство Android (эмулятор или телефон) или iOS
- Собранное приложение FinControl

## Шаг 1: Установка Proxyman

1. Скачайте Proxyman с официального сайта и установите.
2. Запустите приложение. При первом запуске разрешите установку сертификатов (для расшифровки HTTPS).

## Шаг 2: Настройка прокси на устройстве

### Android-эмулятор

1. Узнайте IP вашего компьютера в локальной сети (в Proxyman отображается в подсказках).
2. В настройках эмулятора (Wi‑Fi / Proxy) укажите ручной прокси: **Host** — IP компьютера, **Port** — по умолчанию в Proxyman обычно **9090** (проверьте в **Certificate → Proxy** или настройках).
3. Сохраните настройки.

### Реальное Android-устройство

1. Телефон и компьютер в одной Wi‑Fi сети.
2. В Proxyman: **Tools → Set up Proxy on Android** (или аналог) — там будет указан IP и порт.
3. На телефоне: **Wi‑Fi → текущая сеть → Прокси вручную** — введите этот IP и порт.

### iOS (симулятор или устройство)

- **Симулятор**: трафик симулятора часто идёт через хост. В Proxyman выберите **macOS** (или текущую ОС) как источник и перехватывайте трафик локальных приложений.
- **Устройство**: в Proxyman есть пошаговая настройка для iOS (установка сертификата через описание в меню).

## Шаг 3: Установка сертификата Proxyman на устройстве (HTTPS)

1. В Proxyman откройте раздел установки сертификата на мобильное устройство (например **Certificate → Install on Mobile Device**).
2. На устройстве откройте в браузере указанный URL и установите профиль/сертификат.
3. На Android 7+ для приложений: **Настройки → Безопасность → Установить сертификат** (или «Доверенные учётные данные») и выберите установленный сертификат Proxyman.

## Шаг 4: Включение перехвата HTTPS

1. В Proxyman убедитесь, что включён перехват SSL/TLS для нужных доменов (часто по умолчанию уже включён для всех).
2. При необходимости добавьте домены вручную: `api.exchangerate.host`, `open.er-api.com`.

## Шаг 5: Запуск FinControl и проверка

1. В Proxyman включите запись трафика (если есть кнопка Record/Capture).
2. На устройстве откройте FinControl, перейдите на экран **Обменник** или на главный экран с карточкой курсов.
3. В списке запросов Proxyman должны появиться вызовы к:
   - `api.exchangerate.host` (путь вида `/latest?base=RUB&symbols=USD,EUR`),
   - или `open.er-api.com` (путь `/v6/latest/RUB`).

## Что проверить

- [ ] Proxyman запущен, на устройстве указан правильный прокси (IP + порт).
- [ ] Сертификат Proxyman установлен и доверен на устройстве.
- [ ] В приложении выполнен сценарий с загрузкой курсов (Обменник или главная).
- [ ] В Proxyman виден запрос к API курсов, статус 200, в ответе JSON с курсами.

## Отличия от Charles

- Proxyman часто удобнее для macOS и iOS (нативная интеграция).
- Интерфейс и порты могут отличаться — ориентируйтесь на подсказки в меню **Certificate** и настройках прокси.

## Устранение неполадок

- **Трафик не виден**: проверьте прокси на устройстве и что приложение не использует только кэш (очистите кэш приложения или переустановите).
- **Ошибки SSL**: переустановите и доверьте сертификат, перезапустите приложение.
