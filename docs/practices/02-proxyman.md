# Практика: Proxyman — перехват трафика FinControl

**Одно приложение** FinControl (мобильное или веб). **Один сценарий:** тот же, что в [01-charles.md](01-charles.md): открываешь экран **Обменник** или главный экран с карточкой курсов — в Proxyman смотришь трафик к API курсов (`api.exchangerate.host` или `open.er-api.com`).

## Цель

Настроить Proxyman на компьютере и перехватывать HTTP/HTTPS-запросы FinControl к API курсов валют: увидеть в Proxyman запросы к `api.exchangerate.host` или `open.er-api.com` при открытии экрана «Обменник» или главного экрана с карточкой курсов (мобильное приложение на эмуляторе/телефоне или iOS; при необходимости — веб-сборка в браузере).

## Безопасность: HTTPS и сертификаты (только учебная среда)

Расшифровка HTTPS требует установки корневого сертификата Proxyman на устройство — весь трафик проходит через твой компьютер. Это допустимо **только в учебной среде**: свой проект, своё устройство или эмулятор, без реальных платёжных или личных данных. На устройствах с банковскими приложениями и личными данными не используйте прокси с перехватом HTTPS. После практики удалите сертификат с устройства. Подробнее: [FAQ — Безопасность Charles/Proxyman](../FAQ.md#charlesproxyman-насколько-безопасно-ставить-их-сертификат-на-телефон).

## Ожидаемый результат

- Proxyman запущен, на устройстве/эмуляторе указан прокси (IP компьютера, порт обычно 9090 — проверьте в настройках Proxyman), сертификат Proxyman установлен и доверен.
- В приложении FinControl открыт экран **Обменник** или главный экран с загрузкой курсов.
- В списке запросов Proxyman виден запрос к `api.exchangerate.host` или `open.er-api.com`, ответ **200**, в теле JSON с полями курсов (`rates`, `date` и т.п.).

**Сценарий (тот же, что в [01-charles.md](01-charles.md)):** мобильное приложение FinControl → экран Обменник или главная с карточкой курсов → трафик к API курсов виден в сниффере; при проверке веб-версии — браузер с тем же прокси и сертификатом.

---

## Что понадобится

- Proxyman ([proxyman.io](https://proxyman.io/)) — для macOS или Windows
- Устройство Android (эмулятор или телефон) или iOS (симулятор/устройство)
- Приложение FinControl (уже собранное и установленное по [00-getting-started.md](00-getting-started.md) или [03-android-studio.md](03-android-studio.md))

## Шаг 1: Установка Proxyman

1. Открой [proxyman.io](https://proxyman.io/), скачай версию для своей ОС (macOS или Windows) и установи.
2. Запусти Proxyman. При первом запуске разреши установку сертификатов (для расшифровки HTTPS).

## Шаг 2: Настройка прокси на устройстве

### Android-эмулятор

1. Узнай IP своего компьютера в локальной сети (в Proxyman отображается в подсказках).
2. В настройках эмулятора (Wi‑Fi / Proxy) укажи ручной прокси: **Host** — IP компьютера, **Port** — по умолчанию в Proxyman обычно **9090** (проверь в **Certificate → Proxy** или настройках).
3. Сохрани настройки.

### Реальное Android-устройство

1. Телефон и компьютер в одной Wi‑Fi сети.
2. В Proxyman: **Tools → Set up Proxy on Android** (или аналог) — там будет указан IP и порт.
3. На телефоне: **Wi‑Fi → текущая сеть → Прокси вручную** — введи этот IP и порт.

### iOS (симулятор или устройство)

- **Симулятор**: трафик симулятора часто идёт через хост. В Proxyman выбери **macOS** (или текущую ОС) как источник и перехватывай трафик локальных приложений.
- **Устройство**: в Proxyman есть пошаговая настройка для iOS (установка сертификата через описание в меню).

## Шаг 3: Установка сертификата Proxyman на устройстве (HTTPS)

1. В Proxyman открой раздел установки сертификата на мобильное устройство (например **Certificate → Install on Mobile Device**).
2. На устройстве открой в браузере указанный URL и установи профиль/сертификат.
3. На Android 7+ для приложений: **Настройки → Безопасность → Установить сертификат** (или «Доверенные учётные данные») и выбери установленный сертификат Proxyman.

## Шаг 4: Включение перехвата HTTPS

1. В Proxyman убедись, что включён перехват SSL/TLS для нужных доменов (часто по умолчанию уже включён для всех).
2. При необходимости добавь домены вручную: `api.exchangerate.host`, `open.er-api.com`.

## Шаг 5: Запуск FinControl и проверка

1. В Proxyman включи запись трафика (если есть кнопка Record/Capture).
2. На устройстве открой FinControl, перейди на экран **Обменник** или на главный экран с карточкой курсов.
3. В списке запросов Proxyman должны появиться вызовы к:
   - `api.exchangerate.host` (путь вида `/latest?base=RUB&symbols=USD,EUR`),
   - или `open.er-api.com` (путь `/v6/latest/RUB`).

## Проверка

- [ ] Proxyman запущен, запись/перехват включён (кнопка Record/Capture, если есть).
- [ ] На устройстве/эмуляторе в настройках Wi‑Fi/сети указан ручной прокси: хост = IP компьютера, порт (обычно **9090** — смотрите в Proxyman: Certificate → Proxy или настройки).
- [ ] Сертификат Proxyman установлен на устройстве и помечен как доверенный (Android: Настройки → Безопасность → Доверенные учётные данные).
- [ ] В Proxyman включён перехват SSL/TLS для доменов `api.exchangerate.host`, `open.er-api.com` (или для всех).
- [ ] В FinControl открыт экран **Обменник** или главный экран с карточкой курсов (при необходимости в настройках приложения «Очистить данные», затем снова открыть экран).
- [ ] В списке запросов Proxyman виден запрос к `api.exchangerate.host` или `open.er-api.com`, статус **200**, в ответе JSON с курсами.

## Что показать на экзамене / созвоне

1. Запусти Proxyman, включи перехват.
2. На устройстве/эмуляторе открой FinControl → экран **Обменник**.
3. В Proxyman покажи запрос к `api.exchangerate.host` или `open.er-api.com`: статус 200, JSON с курсами.
4. Покажи Breakpoint: останови ответ, измени курс USD → покажи изменённый курс в приложении.
5. Покажи Network Conditions: включи медленную сеть → покажи поведение приложения (загрузка, таймаут, кэш).
6. Кратко скажи: «Настроил Proxyman как прокси, перехватил HTTPS-трафик, использовал Breakpoint для модификации ответа и проверил поведение при медленной сети.»

## Траблшутинг

- **Трафик не виден** — пошагово: [FAQ — Charles/Proxyman не видит трафик](../FAQ.md#charles--proxyman-не-видит-трафик-приложения-что-проверить): прокси на устройстве (IP и порт), один Wi‑Fi для телефона и ПК, сертификат установлен и доверен, приложение реально запрашивает курсы (Обменник/главная, при необходимости очистить кэш).
- **Ответ не расшифрован / ошибки SSL** — [FAQ — В Charles вижу запрос, но ответ Unknown](../FAQ.md#в-charles-вижу-запрос-но-ответ-unknown-или-не-расшифрован). Переустанови и доверь сертификат Proxyman, перезапусти приложение.
- **Android 14+ не доверяет сертификату** — [FAQ — На Android 14 приложения не доверяют сертификату](../FAQ.md#на-android-14-или-новее-приложения-всё-равно-не-доверяют-сертификату-charlesproxyman).

## Отличия от Charles

- Proxyman часто удобнее для macOS и iOS (нативная интеграция). Порт по умолчанию — обычно **9090** (у Charles — 8888).
- Интерфейс и названия пунктов меню могут отличаться — ориентируйтесь на разделы **Certificate** и настройки прокси в Proxyman.

## Дополнительно: продвинутые функции Proxyman

### Breakpoint — модификация запроса/ответа

Аналог Breakpoints в Charles: остановить запрос или ответ, изменить и отправить дальше.

1. Правый клик на запросе к `api.exchangerate.host` → **Breakpoint** (или **Tools → Breakpoint**).
2. Выбери: **Request** (перед отправкой) и/или **Response** (перед отдачей в приложение).
3. При следующем запросе Proxyman остановит его — редактируй тело, заголовки, статус-код.
4. Нажми **Send** для отправки.

**Сценарии тестирования:**
- Измени курс USD в ответе на `999.99` → проверь отображение в приложении.
- Измени HTTP-статус на `500` → проверь, что приложение переключается на fallback-провайдер.
- Отправь невалидный JSON → проверь, что приложение не падает.

### Map Local — подмена ответа файлом

1. Правый клик на запросе → **Map Local** (или **Tools → Map Local**).
2. Укажи локальный JSON-файл с подготовленными данными.
3. Все последующие запросы к этому URL будут возвращать содержимое файла.

Создай файл `mock_rates.json`:
```json
{"result":"success","base_code":"RUB","rates":{"USD":0.011,"EUR":0.0095}}
```

### Scripting — автоматизация на JavaScript

Proxyman поддерживает скрипты для автоматической модификации запросов/ответов:

1. Правый клик на запросе → **Scripting**.
2. Пример скрипта (изменить курс USD):
   ```javascript
   function onResponse(context, url, request, response) {
     var body = response.body;
     if (body && body.rates) {
       body.rates.USD = 0.001;
     }
     return response;
   }
   ```
3. Скрипт выполняется для каждого подходящего запроса — не нужно останавливаться вручную.

### Network Conditions — имитация медленной сети

1. **Tools → Network Conditions** (или иконка черепахи в панели).
2. Выбери пресет: **3G**, **Edge**, **Very Bad Network** или задай вручную (bandwidth, latency, packet loss).
3. В приложении проверь:
   - Отображается ли индикатор загрузки?
   - Срабатывает ли таймаут (5 секунд)?
   - Показываются ли кэшированные курсы?
   - Нет ли ANR (Application Not Responding)?

### Diff — сравнение запросов

1. Выбери два запроса (Cmd+клик или Ctrl+клик).
2. Правый клик → **Diff** — Proxyman покажет различия в заголовках, теле, параметрах.
3. Полезно для сравнения: запрос до и после изменения кода, разные версии API.

### Repeat — повтор запроса

Правый клик на запросе → **Repeat** — повторить с теми же параметрами. Удобно для проверки стабильности ответов API.

### Export/Import — сохранение сессии

1. **File → Export** — сохрани все перехваченные запросы в файл `.proxyman`.
2. Файл можно приложить к баг-репорту или передать коллеге для анализа.
3. **File → Import** — загрузи ранее сохранённую сессию.

### Фильтрация и поиск

- **Filter bar** — фильтруй по домену, пути, статусу ответа, типу контента.
- Быстрый поиск: введи `exchangerate` в строку фильтра — останутся только запросы к API курсов.
- Фильтр по статусу: `status:500` — только ошибки сервера.
