# 🌐 Практика 02: Proxyman — перехват и анализ трафика FinControl

> Настраиваем Proxyman для перехвата HTTP/HTTPS-трафика приложения FinControl. Изучаем современный и удобный сниффер с нативной поддержкой macOS/iOS, Breakpoints, Map Local, скриптингом на JavaScript и инструментами сравнения запросов.

---

## 🎯 Цель

Настроить Proxyman на компьютере, подключить к нему устройство (эмулятор или телефон), установить SSL-сертификат и **перехватить HTTPS-запросы** приложения FinControl к API курсов валют (`api.exchangerate.host` или `open.er-api.com`). Научиться использовать Breakpoint, Map Local, Scripting, Network Conditions и другие инструменты Proxyman для тестирования.

---

## ✅ Ожидаемый результат

- ✅ Proxyman установлен, запущен, перехват трафика включён
- ✅ На устройстве/эмуляторе настроен ручной прокси (IP компьютера, порт **9090**)
- ✅ Корневой сертификат Proxyman установлен на устройстве и помечен как доверенный
- ✅ Перехват SSL/TLS включён для нужных доменов
- ✅ В FinControl открыт экран **Обменник** или главный экран с карточкой курсов
- ✅ В списке запросов Proxyman виден запрос к `api.exchangerate.host` или `open.er-api.com` со статусом **200** и JSON-ответом с полями `rates`, `date`

Это допустимо **только в учебной среде**: свой проект, своё устройство или эмулятор, без реальных платёжных или личных данных. **Не ставь** сертификат на устройство с банковскими приложениями и личными данными. После практики **удали** сертификат с устройства. Подробнее: [FAQ — Безопасность Charles/Proxyman](../FAQ.md#charlesproxyman-насколько-безопасно-ставить-их-сертификат-на-телефон).

## 📝 Пошаговая инструкция

### Шаг 1: Установка Proxyman

1. Открой сайт [proxyman.io](https://proxyman.io/) и скачай версию для своей ОС (macOS или Windows).
2. Установи приложение стандартным способом:
   - **macOS:** перетащи в Applications
   - **Windows:** запусти установщик
3. Запусти Proxyman. При первом запуске он попросит разрешение на установку корневых сертификатов — **разреши** (нужно для расшифровки HTTPS).

> 💡 Proxyman имеет бесплатную версию с ограничениями — для учебных целей её достаточно.

---

### Шаг 2: Настройка прокси на устройстве

Тебе нужно направить весь трафик устройства через Proxyman. Для этого указываешь IP своего компьютера и порт **9090** (порт Proxyman по умолчанию).

> 📌 Проверь точный порт в Proxyman: **Certificate → Proxy** или в настройках приложения. По умолчанию — **9090**, но может отличаться.

#### Вариант A: Android-эмулятор

1. Узнай IP своего компьютера в локальной сети:
   ```bash
   # Windows
   ipconfig
   # macOS / Linux
   ifconfig | grep "inet "
   ```
   Тебе нужен адрес вида `192.168.X.X` (не `127.0.0.1`).
2. В Proxyman IP и порт обычно отображаются в подсказках при первом запуске или в разделе **Certificate → Install Certificate on Android**.
3. В настройках эмулятора: **Settings → Network & internet → Internet** (или настройки Wi-Fi) → выбери сеть → **Proxy → Manual**:
   - **Proxy hostname**: IP твоего компьютера
   - **Proxy port**: `9090`
4. Сохрани.

#### Вариант B: Реальное Android-устройство

1. Убедись, что телефон и компьютер подключены к **одной Wi-Fi сети**.
2. В Proxyman: **Tools → Set up Proxy on Android** (или **Certificate → Install Certificate on Android**) — Proxyman покажет IP и порт.
3. На телефоне: **Wi-Fi → долгое нажатие по текущей сети → Изменить сеть → Дополнительно → Прокси → Вручную**:
   - **Хост**: IP компьютера (из Proxyman)
   - **Порт**: `9090`
4. Сохрани.

#### Вариант C: iOS-симулятор (macOS)

Трафик iOS-симулятора идёт через хостовую систему (Mac). В Proxyman:

1. Выбери в боковой панели источник трафика — **macOS** (или текущую ОС).
2. Proxyman автоматически перехватывает трафик локальных приложений, включая симулятор.
3. Никаких дополнительных настроек прокси не нужно.

> 💡 Proxyman умеет **автоматически устанавливать сертификат в iOS-симулятор** — это одно из главных преимуществ перед Charles.

#### Вариант D: Реальное iOS-устройство

1. Телефон и Mac в одной Wi-Fi сети.
2. В Proxyman: **Certificate → Install Certificate on iOS → Physical Device** — следуй пошаговым инструкциям.
3. На iPhone: **Настройки → Wi-Fi → (i) рядом с сетью → Прокси → Вручную**:
   - **Сервер**: IP Мака
   - **Порт**: `9090`

---

### Шаг 3: Установка сертификата Proxyman на устройстве (HTTPS)

Без доверенного сертификата Proxyman ты увидишь запросы, но **содержимое HTTPS** будет зашифровано.

#### Android

1. В Proxyman открой раздел установки сертификата: **Certificate → Install Certificate on Android** (или **Tools → Set up Proxy on Android**). Proxyman покажет URL для скачивания сертификата.
2. На устройстве открой **браузер** и перейди по указанному URL.
3. Скачай и установи сертификат.
4. На Android 7+: перейди в **Settings → Security → Encryption & credentials → Install a certificate → CA certificate** и выбери скачанный файл.
5. Убедись, что сертификат Proxyman появился в **Настройки → Безопасность → Доверенные учётные данные → Пользователь**.

#### iOS

1. На устройстве/симуляторе следуй инструкциям из Proxyman: **Certificate → Install Certificate on iOS**.
2. Для **симулятора** — Proxyman может установить сертификат автоматически (кнопка **Install on iOS Simulators**).
3. Для **физического устройства** — открой Safari, перейди по URL из Proxyman, установи профиль.
4. Перейди в **Настройки → Основные → Об этом устройстве → Доверие сертификатов** и **включи** доверие для Proxyman CA.

> 📌 **Важно для iOS:** нужно и **установить** профиль, и **включить доверие** — это два отдельных шага. Без второго HTTPS не расшифруется.

---

### Шаг 4: Включение перехвата HTTPS (SSL/TLS)

1. В Proxyman убедись, что включён перехват SSL/TLS для нужных доменов.
2. Proxyman часто включает перехват для всех доменов по умолчанию. Если нет — добавь домены вручную:
   - Правый клик на запросе → **Enable SSL Proxying**
   - Или через **Tools → SSL Proxying List** → добавь:
     - `api.exchangerate.host`
     - `open.er-api.com`

> 💡 В Proxyman можно включить SSL Proxying для конкретного домена прямо из списка запросов — правый клик → **Enable SSL Proxying**. Это удобнее, чем в Charles.

---

### Шаг 5: Запуск FinControl и проверка трафика

1. Убедись, что Proxyman запущен и перехват включён (кнопка **Record/Capture** на панели, если она есть).
2. На устройстве/эмуляторе открой приложение **FinControl**.
3. Перейди на экран **Обменник** или на **главный экран** (где подгружаются курсы валют).
4. Переключись в Proxyman — в списке запросов (левая панель) должны появиться:
   - `api.exchangerate.host` — запрос вида `/latest?base=RUB&symbols=USD,EUR,...`
   - или `open.er-api.com` — запрос вида `/v6/latest/RUB` (если основной API недоступен)
5. Кликни на запрос — в правой панели увидишь ответ: статус **200**, тело JSON с полями `rates`, `base`, `date`.

> 📌 Если курсы уже закэшированы и запроса нет — в **Настройках** приложения FinControl нажми **«Очистить данные»**, затем снова открой экран Обменник.

## 🔍 Проверка

- [ ] Proxyman запущен, перехват трафика включён
- [ ] На устройстве в настройках Wi-Fi/сети указан ручной прокси: хост = IP компьютера, порт `9090`
- [ ] Сертификат Proxyman установлен на устройстве и помечен как доверенный
- [ ] В Proxyman включён перехват SSL/TLS для доменов `api.exchangerate.host`, `open.er-api.com` (или для всех)
- [ ] В FinControl открыт экран **Обменник** или главный экран с карточкой курсов
- [ ] В списке запросов Proxyman виден запрос к `api.exchangerate.host` или `open.er-api.com`
- [ ] Статус ответа — **200**, тело — JSON с полями `rates`, `date`

## 🎓 Что показать на экзамене

1. **Запусти Proxyman**, убедись, что перехват включён.
2. На устройстве/эмуляторе открой **FinControl → экран Обменник**.
3. В Proxyman **покажи запрос** к `api.exchangerate.host` или `open.er-api.com`: статус **200**, JSON с курсами.
4. **Покажи Breakpoint:** останови ответ, измени курс USD → покажи изменённый курс в приложении.
5. **Покажи Network Conditions:** включи медленную сеть → покажи поведение приложения (загрузка, таймаут, кэш).
6. Кратко прокомментируй: *«Настроил Proxyman как прокси, перехватил HTTPS-трафик, использовал Breakpoint для модификации ответа и проверил поведение при медленной сети.»*

> 💡 **Бонус для высокой оценки:** продемонстрируй скриптинг — автоматическую модификацию ответа через JavaScript, или покажи Diff двух запросов.

## 🛠 Траблшутинг

| Проблема | Решение |
|----------|---------|
| **Трафик не виден в Proxyman** | Проверь пошагово: прокси на устройстве (IP и порт 9090), телефон и ПК в одной Wi-Fi сети, сертификат установлен и доверен, приложение реально запрашивает курсы (Обменник, при необходимости очисти кэш). Подробнее: [FAQ](../FAQ.md#charles--proxyman-не-видит-трафик-приложения-что-проверить) |
| **Ответ не расшифрован / ошибки SSL** | Переустанови и доверь сертификат Proxyman, перезапусти приложение. Подробнее: [FAQ](../FAQ.md#в-charles-вижу-запрос-но-ответ-unknown-или-не-расшифрован) |
| **Android 14+ не доверяет сертификату** | На Android 14 и новее система ограничивает пользовательские CA. Подробнее: [FAQ](../FAQ.md#на-android-14-или-новее-приложения-всё-равно-не-доверяют-сертификату-charlesproxyman) |
| **Порт 9090 занят** | Измени порт в настройках Proxyman и обнови настройки прокси на устройстве |

## 🧪 Практические сценарии тестирования

Ниже — конкретные сценарии тестирования приложения FinControl через Proxyman. Сценарии аналогичны [практике Charles](01-charles.md), но с учётом интерфейса и возможностей Proxyman.

---

### Сценарий 1: Проверка корректности запроса к API курсов

**Цель:** убедиться, что приложение отправляет правильный запрос к API.

1. Убедись, что Proxyman перехватывает трафик.
2. Открой FinControl → экран **Обменник**.
3. В Proxyman найди запрос к `api.exchangerate.host`.
4. **Проверь запрос (вкладка Request):**
   - Метод: `GET`
   - URL: `https://api.exchangerate.host/latest?base=RUB&symbols=USD,EUR,GBP,CHF,CNY,JPY,KZT,TRY,BRL,INR`
   - Заголовки: `Accept: application/json`, `User-Agent: fin_control/1.0`
5. **Проверь ответ (вкладка Response):**
   - Статус: `200`
   - Content-Type: `application/json`
   - Тело: JSON с полем `rates`
6. **Запиши:** какие валюты вернулись, совпадают ли с отображёнными в приложении.

---

### Сценарий 2: Подмена курса через Breakpoint

**Цель:** проверить, как приложение отображает подменённый курс.

1. В Proxyman: правый клик на запросе к `api.exchangerate.host` → **Breakpoint**.
2. Выбери перехват **Response**.
3. В приложении нажми **Обновить** курсы (или перейди на Обменник).
4. Proxyman остановит ответ — откроется окно редактирования.
5. В теле JSON найди `"USD":0.011...` и измени на `"USD":999.99`.
6. Нажми **Send** (или **Execute**).
7. **Проверь в приложении:** курс USD отображается как **999.99**.
8. **Вывод для баг-репорта:** приложение не валидирует адекватность курсов с API.

---

### Сценарий 3: Имитация ошибки сервера (HTTP 500)

**Цель:** проверить обработку ошибок и fallback-механизм.

1. Установи **Breakpoint** на ответ от `api.exchangerate.host`.
2. В приложении обнови курсы.
3. Proxyman остановит ответ — измени **HTTP Status** на `500`.
4. Нажми **Send**.
5. **Проверь в приложении:**
   - Переключилось ли на fallback-провайдер (`open.er-api.com`)?
   - Если оба недоступны — показало ли кэшированные курсы или сообщение об ошибке?
   - Не упало ли приложение?

---

### Сценарий 4: Map Local — фиксированные данные для тестирования

**Цель:** подменить ответ API локальным файлом для предсказуемого тестирования.

1. Создай файл `test_rates.json`:
   ```json
   {
     "motd": {"msg": "test"},
     "success": true,
     "base": "RUB",
     "date": "2026-01-15",
     "rates": {
       "USD": 0.01,
       "EUR": 0.009,
       "GBP": 0.008,
       "CHF": 0.0095,
       "CNY": 0.075,
       "JPY": 1.55,
       "KZT": 5.0,
       "TRY": 0.35,
       "BRL": 0.055,
       "INR": 0.85
     }
   }
   ```
2. В Proxyman: правый клик на запросе к `api.exchangerate.host` → **Map Local** (или **Tools → Map Local**).
3. Укажи путь к файлу `test_rates.json`.
4. В приложении обнови курсы — они загрузятся из твоего файла.
5. **Для баг-хантинга** — попробуй экстремальные значения:

| Значение в JSON | Что проверяем |
|-----------------|---------------|
| `"USD": 0` | Деление на ноль? |
| `"USD": -1` | Отрицательный курс? |
| `"USD": 999999999` | Переполнение в UI? |

---

### Сценарий 5: Scripting — автоматическая модификация через JavaScript

**Цель:** автоматически менять ответ API без ручной остановки (преимущество Proxyman перед Charles).

1. В Proxyman: правый клик на запросе к `api.exchangerate.host` → **Scripting**.
2. Вставь скрипт:
   ```javascript
   function onResponse(context, url, request, response) {
     var body = response.body;
     if (body && body.rates) {
       body.rates.USD = 0.001; // Подменяем курс USD
     }
     return response;
   }
   ```
3. Скрипт выполняется **автоматически** для каждого подходящего запроса — не нужно останавливать вручную.
4. В приложении обнови курсы — курс USD будет подменён на `0.001`.

> 💡 Скриптинг — мощный инструмент. Можно добавлять задержки, менять заголовки, логировать данные. Документация: [proxyman.io/scripting](https://docs.proxyman.io/scripting/snippet-code).

---

### Сценарий 6: Network Conditions — имитация медленной сети

**Цель:** проверить таймаут и UX при плохом интернете.

1. В Proxyman: **Tools → Network Conditions** (или иконка черепахи 🐢 на панели).
2. Выбери пресет:
   - **3G** — типичная мобильная сеть
   - **Edge** — очень медленная сеть
   - **Very Bad Network** — экстремальные условия
   - Или задай вручную: bandwidth, latency, packet loss
3. В приложении обнови курсы.
4. **Проверь:**
   - Показывается ли индикатор загрузки (спиннер)?
   - Не зависает ли UI (можно ли переключать вкладки)?
   - Срабатывает ли таймаут (5 секунд в FinControl)?
   - При таймауте — показываются ли кэшированные курсы?
   - Нет ли ANR (Application Not Responding)?
5. **Выключи Network Conditions** после теста.

---

### Сценарий 7: Diff — сравнение двух запросов

**Цель:** сравнить ответы API при разных условиях.

1. Сделай запрос курсов в нормальных условиях (открой Обменник).
2. Включи Map Local с изменёнными данными и сделай второй запрос.
3. Выбери оба запроса в списке: **Cmd+клик** (macOS) или **Ctrl+клик** (Windows).
4. Правый клик → **Diff** — Proxyman покажет различия в заголовках, теле и параметрах.
5. **Полезно для:** сравнения запросов до и после изменения кода, разных версий API, разных провайдеров (`api.exchangerate.host` vs `open.er-api.com`).

## 🔧 Дополнительно: продвинутые функции Proxyman

### Repeat — повтор запроса

Правый клик на запросе → **Repeat** — повторить с теми же параметрами. Удобно для проверки стабильности ответов API и воспроизведения проблем.

### Export/Import — сохранение сессии

1. **File → Export** — сохрани все перехваченные запросы в файл `.proxyman`.
2. Файл можно приложить к **баг-репорту** или передать коллеге для анализа.
3. **File → Import** — загрузи ранее сохранённую сессию.

> 💡 Экспорт сессии — отличный способ документировать найденные баги. Приложи файл `.proxyman` к баг-репорту, и разработчик увидит точные запросы и ответы.

### Фильтрация и поиск

| Действие | Как сделать |
|----------|-------------|
| Фильтр по домену | Введи `exchangerate` в строку фильтра |
| Фильтр по статусу | Введи `status:500` — только ошибки сервера |
| Фильтр по типу | Выбери фильтр по Content-Type (JSON, HTML и т.д.) |
| Быстрый поиск | Используй **Filter bar** — фильтруй по домену, пути, статусу, типу контента |
