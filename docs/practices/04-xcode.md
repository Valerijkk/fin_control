# Практика: Xcode — установка в симуляторе и логирование

**Одно приложение** FinControl. Задача: открыть проект в Xcode (`ios/Runner.xcworkspace`), запустить в симуляторе iOS и настроить просмотр логов в **Console** — при работе с экранами (Обменник, Портфель, расходы) видеть вывод в реальном времени.

## Цель

Открыть проект FinControl в Xcode, запустить приложение в симуляторе iOS и настроить просмотр логов (Console), чтобы при работе с экранами (Обменник, Портфель, расходы) видеть вывод в реальном времени.

## Ожидаемый результат

- В Xcode открыт **`ios/Runner.xcworkspace`** (не .xcodeproj), выбрана схема **Runner** и симулятор (например iPhone 15, iOS 17).
- Приложение FinControl успешно собирается и запускается в симуляторе; на экране отображается приложение с навигацией.
- В нижней панели Xcode (Debug area) открыта вкладка **Console**; при действиях в приложении в консоли появляются логи (`print`, `debugPrint`, системные сообщения).

---

## Что понадобится

- macOS с установленным Xcode
- Проект FinControl с добавленной платформой iOS (папка `ios/`)

## Шаг 1: Открытие проекта в Xcode

1. В корне проекта FinControl (там, где папки `lib/`, `ios/`, `pubspec.yaml`) открой **именно** **`ios/Runner.xcworkspace`** — **не** `Runner.xcodeproj` (из-за CocoaPods/Flutter нужен workspace).
2. Двойной клик по файлу `Runner.xcworkspace` в папке `ios/` или из терминала в корне проекта:
   ```bash
   open ios/Runner.xcworkspace
   ```
3. Дождись загрузки проекта в Xcode.

## Шаг 2: Выбор симулятора и запуск

1. В Xcode в верхней панели слева выбери схему **Runner** (выпадающий список рядом с кнопкой Run).
2. В том же выпадающем списке выбери **симулятор** (например iPhone 15, iOS 17) — не «Any iOS Device».
3. Нажми **Run** (▶) или сочетание **Cmd + R**.
4. Дождись окончания сборки (при первой сборке Flutter/CocoaPods могут подтянуть зависимости). Приложение установится и запустится в симуляторе; откроется окно симулятора с FinControl.

Альтернатива из терминала (из корня проекта):

```bash
flutter run
```

Выбери iOS-симулятор из списка.

## Шаг 3: Просмотр логов в Xcode

1. Во время работы приложения открой нижнюю панель (Debug area): **View → Debug Area → Activate Console** или **Cmd + Shift + C**.
2. Убедись, что активна вкладка **Console** (внизу панели). В ней отображаются вывод `print`, `debugPrint`, логи нативного кода и Flutter.
3. В приложении на симуляторе выполни действия: открой экран Обменник, Портфель, добавь расход — в консоли должны появляться новые строки.

## Шаг 4: Фильтрация логов

- В поле поиска консоли введи ключевое слово (например название экрана или тег, если он добавлен в коде).
- Для просмотра только ошибок ориентируйтесь на уровни вывода и стек-трейсы.

## Проверка

- [ ] Открыт именно `ios/Runner.xcworkspace`, вверху выбраны схема **Runner** и симулятор (не Any iOS Device).
- [ ] Сборка завершается без ошибок; приложение запускается в симуляторе и показывает главный экран FinControl.
- [ ] В нижней панели Xcode открыта вкладка **Console**; при переходах по экранам и действиях в приложении в консоли появляются логи.

## Траблшутинг

- **Ошибки подписания / Signing** — открой таргет Runner → **Signing & Capabilities**, включи **Automatically manage signing** и выбери свою команду (Apple ID). Для симулятора достаточно бесплатного аккаунта.
- **Сборка не находит Flutter** — выполни из корня проекта `flutter pub get` и открой снова `ios/Runner.xcworkspace`.

## Что показать на экзамене / созвоне

1. Покажи в Xcode открытый `Runner.xcworkspace`, схему Runner и выбранный симулятор.
2. Запусти приложение (Cmd + R), покажи работающее приложение в симуляторе.
3. Открой Console (Cmd + Shift + C), выполни действия в приложении (Обменник, Портфель) — покажи появление логов в консоли.
4. Кратко скажи: «Открыл workspace в Xcode, запустил на симуляторе, в Console вижу логи при действиях в приложении.»

## Дополнительно: инструменты Xcode для тестирования

### Instruments — профилирование производительности

Instruments — мощный инструмент профилирования в Xcode.

1. **Product → Profile** (Cmd + I) — собирает приложение и открывает Instruments.
2. Выбери шаблон:
   - **Time Profiler** — какие методы занимают больше всего CPU.
   - **Allocations** — использование памяти и утечки.
   - **Leaks** — обнаружение утечек памяти.
   - **Network** — сетевая активность.
3. Нажми Record (красная кнопка), выполни сценарий в приложении, нажми Stop.
4. Анализируй результаты: ищи горячие точки (hot spots) в коде.

**Сценарий:** профилируй загрузку экрана Обменник — проверь, нет ли блокирующих операций на main thread.

### View Debugger — 3D-иерархия UI

1. Запусти приложение в симуляторе через Xcode (Cmd + R).
2. **Debug → View Debugging → Capture View Hierarchy** (или кнопка в Debug bar).
3. Откроется 3D-визуализация всех слоёв интерфейса.
4. Кликай на элементы — в инспекторе справа увидишь свойства (frame, constraints, цвет, текст).

**Что проверять:**
- Нет ли невидимых перекрывающих view.
- Правильны ли размеры и позиции элементов.
- Нет ли лишних слоёв (влияет на производительность рендеринга).

### Memory Graph Debugger — утечки памяти

1. Запусти приложение, походи по экранам.
2. В Debug bar нажми **Memory Graph** (иконка с узлами).
3. Xcode покажет граф объектов в памяти с их связями.
4. Фиолетовым отмечены объекты с циклическими ссылками (потенциальные утечки).

### Network Link Conditioner — имитация медленной сети

Встроенный в macOS инструмент для тестирования при плохой сети:

1. Скачай **Additional Tools for Xcode** с developer.apple.com (раздел Downloads → More Downloads).
2. Установи **Hardware IO Tools** — появится **Network Link Conditioner** в System Preferences.
3. Включи и выбери пресет: **3G**, **Edge**, **Very Bad Network**, **100% Loss**.
4. Вся сеть Mac (и симулятора) пойдёт через эти условия.
5. В приложении проверь: курсы грузятся медленно → таймаут → кэш → сообщение об ошибке.

**Важно:** не забудь выключить после тестирования!

### Console.app — расширенные логи

Помимо консоли Xcode, можно использовать системное приложение Console:

1. Открой **Applications → Utilities → Console.app**.
2. Выбери симулятор в левой панели.
3. В строке поиска введи `FinControl` или `flutter`.
4. Увидишь все системные логи, связанные с приложением, включая нативные сообщения iOS.

### Simulator: полезные функции

В меню симулятора:

- **Device → Shake** — имитация встряхивания (если в приложении есть жесты).
- **Features → Toggle Slow Animations** — замедление анимаций для визуальной проверки.
- **Debug → Trigger iCloud Sync** — имитация синхронизации.
- **File → Save Screen** — скриншот симулятора.
- **File → Record Screen** — запись видео (для баг-репортов).
- **Features → Status Bar Override** — изменение статус-бара (время, батарея, оператор) — для красивых скриншотов.

### Размеры экранов и Accessibility

1. В симуляторе: **Window → Physical Size** — реальный размер устройства.
2. Переключай между разными симуляторами (iPhone SE, iPhone 15 Pro Max, iPad) — проверяй адаптивность.
3. **Settings → Accessibility → VoiceOver** — проверь доступность (каждый элемент должен озвучиваться).
4. **Settings → Accessibility → Larger Text** — проверь, не ломается ли вёрстка при увеличенном шрифте.
