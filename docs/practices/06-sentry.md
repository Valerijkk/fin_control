# Практика: Sentry — настройка и проверка в FinControl

## Цель

Создать проект в Sentry, получить DSN, настроить приложение FinControl на отправку событий, проверить появление событий и крашей в кабинете Sentry.

## Что понадобится

- Аккаунт на [sentry.io](https://sentry.io)
- Проект FinControl с подключённым Sentry SDK (пакет `sentry_flutter`)

## Шаг 1: Регистрация и создание проекта в Sentry

1. Зайдите на [sentry.io](https://sentry.io) и войдите или зарегистрируйтесь.
2. Создайте организацию (если ещё нет).
3. **Create Project** → выберите платформу **Flutter**.
4. Укажите имя проекта (например `fin-control`) и организацию.
5. После создания скопируйте **DSN** (строка вида `https://xxx@xxx.ingest.sentry.io/xxx`). Она понадобится для инициализации SDK.

## Шаг 2: Подключение Sentry в приложении

1. В `pubspec.yaml` добавьте зависимость:
   ```yaml
   dependencies:
     sentry_flutter: ^8.0.0  # актуальную версию смотрите на pub.dev
   ```

2. В проекте FinControl DSN задаётся в `lib/config/telemetry.dart` (константа `sentryDsn`). Заполните её и перезапустите приложение — инициализация выполняется в `lib/main.dart`.
   ```dart
   import 'package:sentry_flutter/sentry_flutter.dart';

   Future<void> main() async {
     await SentryFlutter.init(
       (options) {
         options.dsn = 'ВАШ_DSN';
         options.tracesSampleRate = 1.0;
         options.environment = 'development'; // или 'production'
       },
       appRunner: () => runApp(const FinControlRoot()),
     );
   }
   ```

3. Для тестового краша добавьте в настройках (например в Settings) кнопку «Отправить тестовое исключение»:
   ```dart
   Sentry.captureException(Exception('Тестовое исключение из FinControl'));
   ```

## Шаг 3: Сборка и запуск

1. Выполните `flutter pub get`, затем `flutter run`.
2. Откройте приложение и нажмите кнопку отправки тестового исключения (если добавили).
3. Либо спровоцируйте реальную ошибку (например отключите сеть и откройте экран с загрузкой курсов).

## Шаг 4: Проверка в кабинете Sentry

1. В проекте Sentry откройте раздел **Issues** (или **Errors**).
2. Должно появиться новое событие с вашим тестовым исключением или ошибкой сети.
3. Откройте событие: видны стек-трейс, устройство, версия ОС, breadcrumbs — всё, что нужно для отладки стабильности релиза.

## Что проверить (чек-лист)

- [ ] Проект Flutter создан в Sentry, DSN скопирован.
- [ ] В приложении выполнен `SentryFlutter.init` с этим DSN.
- [ ] После действия в приложении (тестовое исключение или реальная ошибка) событие отображается в Sentry.
- [ ] В событии видны контекст, стек и при необходимости breadcrumbs.

## Дополнительно

- **Release health**: в настройках Sentry можно включить отслеживание «здоровья» релиза (частота крашей, затронутые пользователи).
- **Breadcrumbs**: добавление своих записей перед важными действиями помогает воспроизвести сценарий падения.
