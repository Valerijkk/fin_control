# 🛡️ Практика 06: Sentry — мониторинг ошибок в FinControl

> Подключаем Sentry к приложению FinControl, отправляем тестовое исключение и учимся анализировать события: стек-трейсы, breadcrumbs, контекст устройства. Результат — ты видишь в кабинете Sentry полную картину ошибки, как в реальном проекте.

---

## 🎯 Цель

Создать проект в Sentry (платформа Flutter), получить **DSN**, вставить его в единственное место — `lib/config/student_env.dart` — и убедиться, что тестовое исключение из приложения появляется в кабинете Sentry с полным контекстом: стек-трейс, устройство, breadcrumbs.

---

## ✅ Ожидаемый результат

- ✔️ В кабинете [sentry.io](https://sentry.io) создан Flutter-проект для FinControl; **DSN** скопирован
- ✔️ В файле **`lib/config/student_env.dart`** константа `sentryDsn` заполнена твоим DSN; приложение перезапущено
- ✔️ В приложении на экране **Настройки** (шестерёнка в AppBar) появилась кнопка **«Тест Sentry»**

> 📌 **Куда вставлять DSN:** только в [**lib/config/student_env.dart**](../../lib/config/student_env.dart) — константа `sentryDsn`. Сохрани файл и перезапусти приложение — больше ничего не нужно. Подробнее: [STUDENT_ENV.md](../STUDENT_ENV.md).

## 📝 Пошаговая инструкция

### Шаг 1: Регистрация и создание проекта в Sentry

1. Открой [sentry.io](https://sentry.io) и зарегистрируйся (или войди в существующий аккаунт).
2. Создай организацию, если у тебя её ещё нет (Sentry предложит это при первом входе).
3. Нажми **Create Project** → выбери платформу **Flutter**.
4. Укажи имя проекта — например `fin-control` — и выбери организацию.
5. После создания Sentry покажет страницу с инструкцией — тебе нужен только **DSN**. Это строка вида:
   ```
   https://xxxx@xxxx.ingest.sentry.io/xxxx
   ```
6. Скопируй DSN. Если пропустил — найди его позже: **Settings → Projects → fin-control → Client Keys (DSN)**.

> ⚠️ **Безопасность:** не коммить DSN в репозиторий. Вставляй его только в локальный `student_env.dart`; при push оставляй константу пустой. Подробнее: [STUDENT_ENV.md](../STUDENT_ENV.md) (раздел «Безопасность ключей»).

---

### Шаг 2: Вставка DSN в приложение

1. Открой файл [**lib/config/student_env.dart**](../../lib/config/student_env.dart) в любом редакторе.
2. Найди константу **`sentryDsn`** (в начале файла подписано: «ПРАКТИКА 06 — SENTRY»).
3. Вставь скопированный DSN **между кавычками**:
   ```dart
   // ПРАКТИКА 06 — SENTRY
   const String sentryDsn = 'https://xxxx@xxxx.ingest.sentry.io/xxxx';
   ```
4. Сохрани файл.
5. **Перезапусти приложение** — выполни `flutter run` заново. Инициализация Sentry происходит в `lib/main.dart` при старте; при непустом DSN в приложении появится кнопка **«Тест Sentry»** на экране Настройки.

> 💡 Добавлять код инициализации вручную **не нужно** — в FinControl он уже есть и автоматически подставляет DSN из `student_env.dart`. Не дублируй DSN в других файлах.

---

### Шаг 3: Проверка — отправка тестового события

1. Запусти приложение (`flutter run` после сохранения `student_env.dart`).
2. Открой любой экран с верхней панелью (Список, Обменник) и нажми **иконку шестерёнки** справа вверху → откроется экран **Настройки**.
3. Нажми кнопку **«Тест Sentry»**. Событие отправляется в фоне.
4. Перейди в браузер → кабинет Sentry → твой проект → **Issues**. Обнови страницу (F5).
5. В течение **10–60 секунд** (иногда до пары минут) появится новое событие — тестовое исключение из FinControl.
6. Открой событие и изучи:
   - **Stack trace** — путь вызова от кнопки до `Sentry.captureException`
   - **Device** — модель устройства/эмулятора, версия ОС
   - **Breadcrumbs** — цепочка действий перед ошибкой
   - **Tags** — `app_type: educational`
   - **User** — `student`

## 🧪 Практические сценарии

### Сценарий 1: Тестовое исключение и анализ события

1. Запусти приложение с заполненным `sentryDsn`.
2. Перейди в **Настройки** (шестерёнка) → нажми **«Тест Sentry»**.
3. Открой [sentry.io](https://sentry.io) → твой проект → **Issues**.
4. Найди событие `Тестовое исключение из FinControl (Настройки)`.
5. **Что проверить в событии:**

| Раздел | Что увидишь |
|--------|------------|
| **Stack trace** | Путь вызова от кнопки до `Sentry.captureException` |
| **Breadcrumbs** | Цепочка действий: навигация (Welcome → Shell → Settings), загрузка курсов и т.д. |
| **Tags** | `app_type: educational` |
| **User** | `student` |
| **Device** | Модель устройства/эмулятора, версия Android/iOS |
| **App** | Версия приложения |

---

### Сценарий 2: Отслеживание навигации через breadcrumbs

1. Запусти приложение и пройди по экранам: **Welcome → Начать → Список → Обменник → Портфель → Статистика → Настройки**.
2. Нажми **«Тест Sentry»**.
3. В Sentry открой событие → раздел **Breadcrumbs**.
4. **Что увидишь:** цепочку навигации — каждый переход между экранами записан как breadcrumb благодаря `SentryNavigatorObserver`.
5. **Зачем это нужно:** если бы приложение упало на экране Статистика — ты бы в Sentry увидел, что пользователь шёл через Список → Обменник → Портфель → Статистика, и мог бы воспроизвести путь к ошибке.

---

### Сценарий 3: Breadcrumbs от бизнес-логики

1. Запусти приложение → открой **Обменник** → обменяй 5000 RUB → USD.
2. Перейди в **Настройки** → **«Тест Sentry»**.
3. В Sentry в событии → **Breadcrumbs** найди:
   - `Курсы загружены (exchangerate.host)` — категория `api`
   - `Обмен: 5000 RUB → ... USD` — категория `exchange`
4. Если API курсов был недоступен, увидишь warning-breadcrumbs:
   - `exchangerate.host недоступен, переключение на fallback`

---

### Сценарий 4: Фильтры и поиск в Sentry

1. В Sentry → **Issues** → используй фильтры:
   - **По уровню:** `level:error` — только ошибки
   - **По тегу:** `app_type:educational`
   - **По пользователю:** `user:student`
2. Попробуй **Assign** — назначь issue на себя.
3. Попробуй **Resolve** — отметь как исправленное.
4. **Зачем это нужно:** в реальном проекте QA-инженер фильтрует ошибки по версии, устройству, пользователю и назначает разработчику.

## 📚 Дополнительно: углублённая работа с Sentry

### Breadcrumbs — хлебные крошки

Breadcrumbs — цепочка действий пользователя перед ошибкой. Sentry SDK автоматически записывает некоторые (HTTP-запросы, навигация), но можно добавить свои:

```dart
import 'package:sentry_flutter/sentry_flutter.dart';

Sentry.addBreadcrumb(Breadcrumb(
  message: 'Пользователь открыл экран Обменник',
  category: 'navigation',
  level: SentryLevel.info,
));
```

> 💡 **Зачем:** при падении в Sentry увидишь полную цепочку действий — «открыл Обменник → нажал Конвертировать → ввёл сумму → ошибка». Это критически важно для воспроизведения бага.

---

### Контекст пользователя

Добавь информацию о пользователе (анонимную для учебного проекта):

```dart
Sentry.configureScope((scope) {
  scope.setUser(SentryUser(
    id: 'student-001',
    username: 'test_student',
  ));
  scope.setTag('screen', 'exchange');
  scope.setExtra('portfolio_balance', 100000.0);
});
```

В событии Sentry увидишь: кто столкнулся с ошибкой, на каком экране, с какими данными.

---

### Performance Monitoring — трейсы

Sentry SDK поддерживает трейсы производительности (`tracesSampleRate = 1.0` уже установлен в FinControl):

```dart
final transaction = Sentry.startTransaction('loadRates', 'task');
try {
  await RatesApi.fetch();
  transaction.status = const SpanStatus.ok();
} catch (e) {
  transaction.status = const SpanStatus.internalError();
  transaction.throwable = e;
} finally {
  await transaction.finish();
}
```

В Sentry → **Performance** увидишь время выполнения операций — можно искать регрессии производительности.

---

### Release Tracking — отслеживание версий

В инициализации можно указать версию:

```dart
options.release = 'fin_control@1.0.0+1';
options.dist = '1';
```

В Sentry → **Releases** увидишь: какая версия стабильна, в какой больше ошибок, adoption rate.

### Работа с Issues в Sentry UI

| Действие | Описание |
|----------|----------|
| **Фильтры** | По уровню (`error`, `fatal`, `warning`), по тегам, по устройству |
| **Assign** | Назначь issue на себя или коллегу |
| **Merge** | Объедини дубликаты одной и той же ошибки |
| **Resolve** | Отметь как исправленное; если повторится — Sentry автоматически переоткроет |
| **Ignore** | Игнорировать известную ошибку (не спамить уведомлениями) |

### Alerts — уведомления

1. В Sentry: **Settings → Alerts** → **Create Alert Rule**.
2. Настрой: «при появлении нового issue уровня error → отправить email/Slack».
3. Для учебного проекта — настрой email-уведомление и покажи его на экзамене.
