# Практика: Sentry — настройка и проверка в FinControl

**Одно приложение** FinControl. Всё из коробки: нужно только **вставить DSN в `lib/config/student_env.dart`** и перезапустить приложение. Проверка за **1–2 действия:** Настройки → кнопка «Тест Sentry» → в кабинете Sentry (Issues) появляется событие.

## Цель

Создать один проект в Sentry для приложения FinControl, получить DSN, вставить его в единственное место в коде — `lib/config/student_env.dart` — и проверить отправку событий: нажать кнопку «Тест Sentry» в настройках приложения и убедиться, что событие появилось в кабинете Sentry (Issues).

## Ожидаемый результат

- В кабинете [sentry.io](https://sentry.io) создан проект (платформа Flutter) для приложения FinControl; DSN скопирован.
- В файле **`lib/config/student_env.dart`** в константу `sentryDsn` вставлен ваш DSN (между кавычками); приложение перезапущено.
- В приложении FinControl в экране **Настройки** (иконка шестерёнки в AppBar любого экрана) доступна кнопка **«Тест Sentry»**; после нажатия в кабинете Sentry в разделе **Issues** появляется событие с тестовым исключением (в течение 10–60 секунд). В событии видны стек-трейс, устройство, breadcrumbs.

**Куда вставлять DSN:** только в [**lib/config/student_env.dart**](../../lib/config/student_env.dart) — константа `sentryDsn`. Сохрани файл и перезапусти приложение — больше ничего не нужно. Подробнее: [STUDENT_ENV.md](../STUDENT_ENV.md).

## Что понадобится

- Аккаунт на [sentry.io](https://sentry.io)
- Проект FinControl с подключённым Sentry SDK (пакет `sentry_flutter`)

## Шаг 1: Регистрация и создание проекта в Sentry

1. Зайди на [sentry.io](https://sentry.io) и войди или зарегистрируйся.
2. Создай организацию (если ещё нет).
3. **Create Project** → выбери платформу **Flutter**.
4. Укажи имя проекта (например `fin-control`) и организацию.
5. После создания скопируй **DSN** (строка вида `https://xxx@xxx.ingest.sentry.io/xxx`). Она понадобится для вставки в `student_env.dart`.

**Безопасность:** не коммитьте DSN и конфиги с ключами в репозиторий. Вставляйте DSN только в локальный `student_env.dart`; при push оставляйте константу пустой. См. [STUDENT_ENV.md](../STUDENT_ENV.md) (раздел «Безопасность ключей»).

## Шаг 2: Куда вставить DSN в приложении

1. В проекте FinControl DSN задаётся **в одном файле**. Открой [**lib/config/student_env.dart**](../../lib/config/student_env.dart).
2. Найди константу **`sentryDsn`** (в начале файла капсом подписано: «ПРАКТИКА 06 — SENTRY»). Вставь скопированный DSN **в пустую строку между кавычками**, например:
   ```dart
   const String sentryDsn = 'https://xxxx@xxxx.ingest.sentry.io/xxxx';
   ```
3. Сохрани файл. **Перезапусти приложение** (`flutter run` заново) — инициализация Sentry выполняется в `lib/main.dart` при старте; при непустом DSN в приложении появится кнопка **«Тест Sentry»** в экране Настройки (иконка шестерёнки в правом верхнем углу любого экрана с AppBar).
4. Добавлять код инициализации вручную не нужно — в FinControl он уже есть и подставляет DSN из `student_env.dart`. Не дублируй DSN в других файлах.

## Шаг 3: Проверка за 1–2 действия

1. Запусти приложение (`flutter run` после сохранения `student_env.dart`).
2. В приложении открой любой экран с верхней панелью (Список или Обменник) и нажми **иконку шестерёнки** справа вверху → экран **Настройки**.
3. Нажми кнопку **«Тест Sentry»**. Событие уходит в фоне.
4. В браузере открой кабинет Sentry → твой проект → **Issues**. Обнови страницу (F5). В течение **10–60 секунд** (иногда до пары минут) появится новое событие с тестовым исключением из FinControl.
5. Открой событие: видны стек-трейс, устройство, версия ОС, breadcrumbs — всё, что нужно для отладки.

## Проверка

- [ ] Проект Flutter создан в Sentry, DSN скопирован из Settings → Client Keys (DSN).
- [ ] В **`lib/config/student_env.dart`** в `sentryDsn` вставлен ваш DSN (не пустая строка); файл сохранён, приложение перезапущено.
- [ ] В приложении в Настройках видна кнопка **«Тест Sentry»**; после нажатия событие появляется в Sentry → Issues.
- [ ] В открытом событии видны контекст, стек и при необходимости breadcrumbs.

## Что показать на экзамене / созвоне

1. Открыть приложение FinControl → **Настройки** (шестерёнка в AppBar) → нажать **«Тест Sentry»**.
2. Открыть кабинет Sentry → **Issues** — показать появившееся событие.
3. Открыть событие: видны стек-трейс, устройство, breadcrumbs.
4. Кратко сказать: «Подключил Sentry по DSN в student_env.dart, в приложении нажал «Тест Sentry» — событие появилось в кабинете с контекстом и стеком.»

## Траблшутинг

- **Кнопка «Тест Sentry» не появляется** — проверь, что в `student_env.dart` в `sentryDsn` вставлен непустой DSN и приложение перезапущено после сохранения файла. [FAQ — Кнопка «Тест Sentry» не отправляет событие](../FAQ.md#кнопка-тест-sentry-в-настройках-не-отправляет-событие).
- **Событие не появляется в Sentry** — задержка 10–60 секунд нормальна; обнови страницу Issues (F5). Если за 2–3 минуты события нет — проверь DSN и перезапуск. [FAQ — Событие в Sentry не появляется сразу](../FAQ.md#событие-в-sentry-не-появляется-сразу).
- **Где задаётся DSN** — только в [STUDENT_ENV.md](../STUDENT_ENV.md) (файл `lib/config/student_env.dart`). Не коммить реальный DSN в репозиторий: [FAQ — Безопасность ключей](../FAQ.md#можно-ли-коммитить-в-git-файл-student_envdart-с-вставленными-dsn-и-api-key).

## Практические сценарии работы с Sentry в FinControl

### Сценарий 1: Тестовое исключение и анализ события

1. Запусти приложение с заполненным `sentryDsn`.
2. Перейди в **Настройки** (шестерёнка) → нажми **Тест Sentry**.
3. Открой [sentry.io](https://sentry.io) → твой проект → **Issues**.
4. Найди событие `Тестовое исключение из FinControl (Настройки)`.
5. **Что проверить в событии:**
   - **Stack trace** — путь вызова от кнопки до `Sentry.captureException`.
   - **Breadcrumbs** — цепочка действий: навигация (Welcome → Shell → Settings), загрузка курсов, и т.д.
   - **Tags** — `app_type: educational`.
   - **User** — `student`.
   - **Device** — модель устройства/эмулятора, версия Android/iOS.
   - **App** — версия приложения.

### Сценарий 2: Отслеживание навигации через breadcrumbs

1. Запусти приложение → навигируй: **Welcome → Начать → Список → Обменник → Портфель → Статистика → Настройки**.
2. Нажми **Тест Sentry**.
3. В Sentry открой событие → раздел **Breadcrumbs**.
4. **Что увидишь:** цепочку навигации — каждый переход между экранами записан как breadcrumb благодаря `SentryNavigatorObserver`.
5. **Зачем это нужно:** если бы приложение упало на экране Статистика — ты бы в Sentry увидел, что пользователь шёл через Список → Обменник → Портфель → Статистика, и мог бы воспроизвести путь.

### Сценарий 3: Breadcrumbs от бизнес-логики

1. Запусти приложение → открой **Обменник** → обменяй 5000 RUB → USD.
2. Перейди в **Настройки** → **Тест Sentry**.
3. В Sentry в событии → **Breadcrumbs** найди:
   - `Курсы загружены (exchangerate.host)` — категория `api`
   - `Обмен: 5000 RUB → ... USD` — категория `exchange`
4. Если API курсов был недоступен, увидишь warning breadcrumbs:
   - `exchangerate.host недоступен, переключение на fallback`

### Сценарий 4: Фильтры и поиск в Sentry

1. В Sentry → **Issues** → используй фильтры:
   - **По уровню:** `level:error` — только ошибки.
   - **По тегу:** `app_type:educational`.
   - **По пользователю:** `user:student`.
2. Попробуй **Assign** — назначь issue на себя.
3. Попробуй **Resolve** — отметь как исправленное.
4. **Зачем это нужно:** в реальном проекте QA-инженер фильтрует ошибки по версии, устройству, пользователю и назначает разработчику.

## Дополнительно: углублённая работа с Sentry

### Breadcrumbs — хлебные крошки

Breadcrumbs — цепочка действий пользователя перед ошибкой. Sentry SDK автоматически записывает некоторые (HTTP-запросы, навигация), но можно добавить свои:

```dart
import 'package:sentry_flutter/sentry_flutter.dart';

Sentry.addBreadcrumb(Breadcrumb(
  message: 'Пользователь открыл экран Обменник',
  category: 'navigation',
  level: SentryLevel.info,
));
```

**Зачем:** при падении в Sentry увидишь полную цепочку действий — «открыл Обменник → нажал Конвертировать → ввёл сумму → ошибка». Это критически важно для воспроизведения бага.

### Контекст пользователя

Добавь информацию о пользователе (анонимную для учебного проекта):

```dart
Sentry.configureScope((scope) {
  scope.setUser(SentryUser(
    id: 'student-001',
    username: 'test_student',
  ));
  scope.setTag('screen', 'exchange');
  scope.setExtra('portfolio_balance', 100000.0);
});
```

В событии Sentry увидишь: кто столкнулся с ошибкой, на каком экране, с какими данными.

### Performance Monitoring — трейсы

Sentry SDK поддерживает трейсы производительности (tracesSampleRate = 1.0 уже установлен в FinControl):

```dart
final transaction = Sentry.startTransaction('loadRates', 'task');
try {
  await RatesApi.fetch();
  transaction.status = const SpanStatus.ok();
} catch (e) {
  transaction.status = const SpanStatus.internalError();
  transaction.throwable = e;
} finally {
  await transaction.finish();
}
```

В Sentry → **Performance** увидишь время выполнения операций, можно искать регрессии.

### Release Tracking — отслеживание версий

В инициализации добавь версию:

```dart
options.release = 'fin_control@1.0.0+1';
options.dist = '1';
```

В Sentry → **Releases** увидишь: какая версия стабильна, в какой версии больше ошибок, adoption rate.

### Фильтры и поиск в Sentry UI

В кабинете Sentry:
- **Issues** → фильтр по уровню (`error`, `fatal`, `warning`).
- **Assign** — назначь issue на себя для отслеживания.
- **Merge** — объедини дубликаты одной и той же ошибки.
- **Resolve** → ошибка исправлена; если повторится — Sentry автоматически переоткроет.
- **Ignore** — игнорировать известную ошибку (не спамить уведомлениями).

### Alerts — уведомления

1. **Settings → Alerts** → **Create Alert Rule**.
2. Настрой: «при появлении нового issue уровня error → отправить email/Slack».
3. Для учебного проекта — настрой email-уведомление и покажи его на экзамене.
