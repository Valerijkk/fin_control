# Практика: Charles Proxy — перехват трафика FinControl

**Одно приложение** FinControl (мобильное или веб). **Один сценарий:** запускаешь приложение, открываешь экран **Обменник** или главный экран с карточкой курсов — в Charles смотришь трафик к API курсов (`api.exchangerate.host` или `open.er-api.com`). Веб-версия: тот же прокси и сертификат в браузере.

## Цель

Настроить Charles Proxy на компьютере и перехватывать HTTP/HTTPS-запросы FinControl к API курсов валют: увидеть в Charles запросы к `api.exchangerate.host` или `open.er-api.com` при открытии экрана «Обменник» или главного экрана с карточкой курсов (мобильное приложение на эмуляторе/телефоне; при необходимости — веб-сборка в браузере).

## Безопасность: HTTPS и сертификаты (только учебная среда)

Перехват HTTPS возможен только при установке на устройство корневого сертификата Charles — трафик расшифровывается на твоём компьютере. Это допустимо **только в учебной среде**: свой проект, своё устройство или эмулятор, без реальных платёжных или личных данных. На устройствах с банковскими приложениями, почтой и соцсетями не используйте прокси с перехватом HTTPS и не доверяйте таким CA. После практики удалите сертификат с устройства (Настройки → Безопасность → Учётные данные). Подробнее: [FAQ — Безопасность Charles/Proxyman](../FAQ.md#charlesproxyman-насколько-безопасно-ставить-их-сертификат-на-телефон).

## Ожидаемый результат

- Charles запущен, на устройстве/эмуляторе указан прокси (IP компьютера, порт 8888), сертификат Charles установлен и доверен.
- В приложении FinControl открыт экран **Обменник** или главный экран с загрузкой курсов.
- В Charles в дереве запросов виден запрос к `api.exchangerate.host` или `open.er-api.com`, ответ **200**, в теле JSON с полями курсов (`rates`, `date` и т.п.).

**Сценарий (тот же для Proxyman, см. [02-proxyman.md](02-proxyman.md)):** мобильное приложение FinControl → экран Обменник или главная с карточкой курсов → трафик к API курсов виден в сниффере; при проверке веб-версии — браузер с тем же прокси и сертификатом.

## Что понадобится

- Charles Proxy ([charlesproxy.com](https://www.charlesproxy.com/))
- Устройство Android (эмулятор или телефон) или iOS-симулятор/устройство
- Приложение FinControl (уже собранное и установленное по [00-getting-started.md](00-getting-started.md) или [03-android-studio.md](03-android-studio.md))

## Шаг 1: Установка Charles

1. Открой [charlesproxy.com](https://www.charlesproxy.com/), скачай установщик для своей ОС и установи программу.
2. Запусти Charles. При первом запуске при запросе разреши установку сертификата Charles в систему (нужно для расшифровки HTTPS на этом компьютере).

## Шаг 2: Настройка прокси на устройстве/эмуляторе

### Android-эмулятор

1. В эмуляторе: **Settings → Network & internet → Internet** (или **Proxy** в настройках Wi‑Fi).
2. Укажи ручной прокси: **Host** — IP твоего компьютера в локальной сети (например `192.168.1.100`), **Port** — обычно `8888` (порт Charles по умолчанию).
3. Альтернатива: в AVD можно задать `-http-proxy http://<твой_IP>:8888` при запуске эмулятора.

### Реальное Android-устройство

1. Убедись, что телефон и компьютер в одной Wi‑Fi сети.
2. В Charles: **Proxy → Proxy Settings** — порт 8888.
3. На телефоне: **Wi‑Fi → долгое нажатие по сети → Изменить / Дополнительно → Прокси вручную** — укажи IP компьютера и порт 8888.

### iOS-симулятор

Симулятор использует сеть Mac/ПК. В Charles настрой **Proxy → macOS Proxy** (или Windows Proxy), чтобы перехватывать трафик приложений на том же компьютере.

## Шаг 3: Установка сертификата Charles на устройстве (HTTPS)

Без доверенного сертификата Charles ты не увидишь расшифрованный HTTPS.

### Android

1. В Charles: **Help → SSL Proxying → Install Charles Root Certificate on a Mobile Device or Remote Browser**.
2. Открой на устройстве браузер и перейди по указанному адресу (например `chls.pro/ssl`).
3. Скачай и установи сертификат. В Android 7+ для приложений может потребоваться поместить сертификат в «Доверенные учётные данные» (Settings → Security → Encryption & credentials → Install a certificate).

### iOS

1. На устройстве/симуляторе открой в Safari адрес из Charles (**Help → SSL Proxying → Install Charles Root Certificate…**).
2. Установи профиль, затем **Настройки → Основные → Об этом устройстве → Доверие** — включи доверие для Charles.

## Шаг 4: SSL Proxying в Charles

1. **Proxy → SSL Proxying Settings**.
2. Включи **Enable SSL Proxying**, добавь **Host** `*` и **Port** `443` (или конкретные хосты: `api.exchangerate.host`, `open.er-api.com`).

## Шаг 5: Запуск FinControl и проверка трафика

1. Запусти Charles, включи запись (**Proxy → Start Recording**).
2. На устройстве/эмуляторе открой приложение FinControl.
3. Перейди на экран **Обменник** или на главный экран (где подгружаются курсы валют).
4. В Charles в дереве запросов должны появиться вызовы к:
   - `api.exchangerate.host` (например `https://api.exchangerate.host/latest?base=RUB&symbols=USD,EUR`),
   - или `open.er-api.com` (fallback, например `https://open.er-api.com/v6/latest/RUB`).

## Проверка

- [ ] Charles запущен, запись включена (Proxy → Start Recording).
- [ ] На устройстве/эмуляторе в настройках Wi‑Fi/сети указан ручной прокси: хост = IP компьютера, порт 8888.
- [ ] Сертификат Charles установлен на устройстве и помечен как доверенный (Android: Настройки → Безопасность → Доверенные учётные данные).
- [ ] В Charles включён SSL Proxying (Proxy → SSL Proxying Settings): Enable SSL Proxying, добавлен хост `*` и порт 443 или хосты `api.exchangerate.host`, `open.er-api.com`.
- [ ] В FinControl открыт экран **Обменник** или главный экран с карточкой курсов (если курсы уже в кэше — в настройках приложения можно «Очистить данные», затем снова открыть экран).
- [ ] В дереве запросов Charles виден запрос к `api.exchangerate.host` или `open.er-api.com`, статус ответа **200**, в теле JSON с полями `rates`, `date` или аналог.

## Траблшутинг

- **Трафик не появляется в Charles** — пошагово: [FAQ — Charles/Proxyman не видит трафик](../FAQ.md#charles--proxyman-не-видит-трафик-приложения-что-проверить): прокси на устройстве (IP и порт 8888), один Wi‑Fi для телефона и ПК, сертификат установлен и доверен, SSL Proxying включён, приложение реально запрашивает курсы (экран Обменник/главная, при необходимости очистить кэш).
- **Ответ «Unknown» или не расшифрован** — [FAQ — В Charles вижу запрос, но ответ Unknown](../FAQ.md#в-charles-вижу-запрос-но-ответ-unknown-или-не-расшифрован).
- **Порт 8888 занят** — [FAQ — Порт 8888 занят](../FAQ.md#порт-8888-charles-занят-или-address-already-in-use).
- **Веб-сборка:** если проверяете FinControl в браузере — настройте прокси браузера на `localhost:8888` и установите сертификат Charles в браузере (Charles подскажет URL).

## Что показать на экзамене / созвоне

1. Запусти Charles, убедись, что запись включена.
2. На устройстве/эмуляторе открой FinControl → экран **Обменник** (или главный с карточкой курсов).
3. В Charles покажи дерево запросов: есть запрос к `api.exchangerate.host` или `open.er-api.com`.
4. Открой ответ: HTTP 200, в теле JSON с полями курсов (rates).
5. Кратко скажи: «Настроил прокси на устройстве, установил и доверил сертификат Charles. При открытии курсов в приложении запрос к API виден в Charles, HTTPS расшифровывается.»

## Дополнительно: продвинутые функции Charles

### Breakpoints — изменение запроса/ответа на лету

Breakpoints позволяют остановить запрос или ответ, изменить его и отправить дальше. Это ключевой навык для тестирования: что будет, если API вернёт ошибку? Что если изменить курс в ответе?

1. В Charles: **Proxy → Breakpoint Settings** → **Enable Breakpoints**.
2. Добавь правило: **Host** = `api.exchangerate.host`, **Path** = `*`, **Query** = `*`.
3. Выбери, на чём останавливаться: **Request** (перед отправкой на сервер) и/или **Response** (перед отдачей в приложение).
4. В приложении открой Обменник → запрос остановится.
5. В окне Breakpoint:
   - **Для Response:** измени тело JSON — например поставь `"USD": 999.99` в объекте `rates`. Нажми **Execute**.
   - **Для Request:** можно изменить параметры запроса (например `symbols=USD,EUR,GBP` → `symbols=USD`).
6. Проверь в приложении: курс USD отобразился как 999.99 (или с ограниченным набором валют).

**Сценарий для тестирования:**
- Изменить статус ответа на `500` → проверить, что приложение показывает ошибку или переключается на fallback-провайдер.
- Изменить тело ответа на невалидный JSON → проверить, что приложение не падает.
- Убрать поле `rates` из ответа → проверить обработку некорректных данных.

### Map Remote — подмена URL

Map Remote перенаправляет запросы с одного URL на другой. Полезно для тестирования с другим API или для подмены production-API на тестовый.

1. **Tools → Map Remote** → **Enable Map Remote** → **Add**.
2. **Map From:** `https://api.exchangerate.host/latest*`
3. **Map To:** `https://open.er-api.com/v6/latest/RUB` (или свой мок-сервер).
4. Теперь все запросы к основному API будут уходить на fallback — проверь, что приложение работает корректно.

### Map Local — подмена ответа файлом

Map Local возвращает ответ из локального файла вместо реального запроса. Идеально для воспроизведения конкретных сценариев.

1. Создай файл `mock_rates.json` на компьютере:
   ```json
   {"result":"success","base_code":"RUB","rates":{"USD":0.011,"EUR":0.0095,"GBP":0.008}}
   ```
2. **Tools → Map Local** → **Enable Map Local** → **Add**.
3. **Host:** `api.exchangerate.host`, **Path:** `/latest*`.
4. **Local path:** укажи путь к `mock_rates.json`.
5. Открой приложение → курсы загрузятся из локального файла.

**Сценарии:**
- Подготовь файл с экстремальными курсами (0.0001, 99999.99) — проверь отображение.
- Подготовь пустой ответ `{}` — проверь обработку.
- Подготовь ответ с отсутствующими полями — проверь устойчивость парсера.

### Rewrite — автоматическая модификация

Rewrite автоматически модифицирует запросы/ответы по правилам (без остановки, в отличие от Breakpoints).

1. **Tools → Rewrite** → **Enable Rewrite** → **Add**.
2. Добавь Location: `api.exchangerate.host`.
3. Добавь правило:
   - **Type:** Body → **Match:** `"USD":` → **Replace:** `"USD":0.001,` — все ответы будут содержать изменённый курс USD.
   - **Type:** Header → добавить заголовок `X-Test: true` ко всем запросам.

### Throttling — имитация медленной сети

Проверка поведения при плохом интернете — критически важно для мобильного тестирования.

1. **Proxy → Throttle Settings**.
2. Включи **Enable Throttling**, выбери пресет:
   - **56 kbps Modem** — очень медленная сеть
   - **3G** — типичная мобильная сеть
   - **Custom:** задай свои значения Bandwidth, Latency, Reliability.
3. Применяй только к нужным хостам (чекбокс **Only for selected hosts** → добавь `api.exchangerate.host`).
4. В приложении открой Обменник — курсы должны грузиться медленно. Проверь:
   - Есть ли индикатор загрузки?
   - Не зависает ли UI?
   - Срабатывает ли таймаут (5 секунд в FinControl)?
   - Показываются ли кэшированные данные при таймауте?

### Repeat & Repeat Advanced

- **Правый клик на запросе → Repeat** — повторить запрос один раз.
- **Repeat Advanced** — повторить N раз с задержкой (нагрузочное тестирование).

Пример: повтори запрос курсов 50 раз с задержкой 100 мс — проверь, что API не блокирует и приложение корректно обрабатывает все ответы.

### No Caching

**Tools → No Caching** — отключает кэширование на стороне клиента (убирает заголовки `If-Modified-Since`, `If-None-Match`). Полезно, чтобы каждый запрос возвращал свежие данные.

### Compose — создание запроса вручную

**Tools → Compose** — создай HTTP-запрос с нуля и отправь на сервер. Полезно для тестирования API без приложения:

1. **Method:** GET
2. **URL:** `https://api.exchangerate.host/latest?base=RUB&symbols=USD,EUR`
3. Добавь заголовки: `Accept: application/json`, `User-Agent: fin_control/1.0`
4. **Execute** — увидишь ответ в Charles.
