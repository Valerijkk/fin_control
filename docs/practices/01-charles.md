# 🔍 Практика 01: Charles Proxy — перехват и анализ трафика FinControl

> Настраиваем Charles Proxy для перехвата HTTP/HTTPS-трафика приложения FinControl. Учимся читать запросы к API курсов валют, модифицировать ответы на лету, имитировать ошибки сервера и медленную сеть — всё, что нужно для профессионального тестирования мобильных приложений.

---

## 🎯 Цель

Настроить Charles Proxy на компьютере, подключить к нему устройство (эмулятор или телефон), установить SSL-сертификат и **перехватить HTTPS-запросы** приложения FinControl к API курсов валют (`api.exchangerate.host` или `open.er-api.com`). Научиться использовать Breakpoints, Map Local, Throttling и другие инструменты Charles для тестирования.

---

## ✅ Ожидаемый результат

- ✅ Charles Proxy установлен, запущен, запись трафика включена
- ✅ На устройстве/эмуляторе настроен ручной прокси (IP компьютера, порт **8888**)
- ✅ Корневой сертификат Charles установлен на устройстве и помечен как доверенный
- ✅ SSL Proxying включён для нужных доменов
- ✅ В FinControl открыт экран **Обменник** или главный экран с карточкой курсов
- ✅ В дереве запросов Charles виден запрос к `api.exchangerate.host` или `open.er-api.com` со статусом **200** и JSON-ответом с полями `rates`, `date`

---

> ⚠️ **Безопасность: только учебная среда!** Перехват HTTPS возможен только при установке корневого сертификата Charles на устройство — весь HTTPS-трафик расшифровывается на твоём компьютере. Это допустимо **только в учебной среде**: свой проект, своё устройство или эмулятор, без реальных платёжных или личных данных. **Не ставь** сертификат Charles на устройство, где есть банковские приложения, почта или соцсети. После практики **удали** сертификат (Настройки → Безопасность → Учётные данные). Подробнее: [FAQ — Безопасность Charles/Proxyman](../FAQ.md#charlesproxyman-насколько-безопасно-ставить-их-сертификат-на-телефон).

## 📝 Пошаговая инструкция

### Шаг 1: Установка Charles Proxy

1. Открой сайт [charlesproxy.com](https://www.charlesproxy.com/) и скачай установщик для своей ОС (Windows / macOS / Linux).
2. Установи программу стандартным способом (Next → Next → Install).
3. Запусти Charles. При первом запуске он попросит разрешение на установку корневого сертификата в систему — **разреши** (это нужно для расшифровки HTTPS на этом компьютере).

> 💡 Бесплатная версия Charles работает 30 дней, но периодически перезапускается с задержкой — для учёбы этого достаточно.

---

### Шаг 2: Настройка прокси на устройстве/эмуляторе

Тебе нужно направить весь трафик устройства через Charles. Для этого указываешь IP своего компьютера и порт **8888** (порт Charles по умолчанию).

#### Вариант A: Android-эмулятор

1. Узнай IP своего компьютера в локальной сети:
   ```bash
   # Windows
   ipconfig
   # macOS / Linux
   ifconfig | grep "inet "
   ```
   Тебе нужен адрес вида `192.168.X.X` (не `127.0.0.1`).
2. В эмуляторе открой **Settings → Network & internet → Internet** (или настройки Wi-Fi).
3. Выбери текущую сеть → **Proxy → Manual**:
   - **Proxy hostname**: IP твоего компьютера (например `192.168.1.100`)
   - **Proxy port**: `8888`
4. Сохрани.

> 💡 **Альтернатива:** при запуске AVD через командную строку можно указать прокси сразу:
> ```bash
> emulator -avd <имя_AVD> -http-proxy http://192.168.1.100:8888
> ```

#### Вариант B: Реальное Android-устройство

1. Убедись, что телефон и компьютер подключены к **одной Wi-Fi сети**.
2. В Charles: **Proxy → Proxy Settings** — убедись, что порт **8888**.
3. На телефоне: **Wi-Fi → долгое нажатие по текущей сети → Изменить сеть → Дополнительно → Прокси → Вручную**:
   - **Хост**: IP компьютера
   - **Порт**: `8888`
4. Сохрани. В Charles появится диалог «Разрешить подключение с ...» — нажми **Allow**.

#### Вариант C: iOS-симулятор

iOS-симулятор использует сеть Mac напрямую. В Charles включи **Proxy → macOS Proxy** — этого достаточно, чтобы перехватывать трафик симулятора.

#### Вариант D: Реальное iOS-устройство

1. Телефон и Mac в одной Wi-Fi сети.
2. На iPhone: **Настройки → Wi-Fi → (i) рядом с сетью → Прокси → Вручную**:
   - **Сервер**: IP Мака
   - **Порт**: `8888`

---

### Шаг 3: Установка сертификата Charles на устройстве (HTTPS)

Без доверенного сертификата Charles ты увидишь запросы, но **содержимое HTTPS** будет зашифровано (статус «Unknown»).

#### Android

1. В Charles: **Help → SSL Proxying → Install Charles Root Certificate on a Mobile Device or Remote Browser** — Charles покажет адрес (обычно `chls.pro/ssl`).
2. На устройстве открой **браузер** и перейди по адресу `chls.pro/ssl`.
3. Скачается файл сертификата — установи его.
4. На Android 7+: перейди в **Settings → Security → Encryption & credentials → Install a certificate → CA certificate** и выбери скачанный файл.
5. Убедись, что сертификат появился в **Настройки → Безопасность → Доверенные учётные данные → Пользователь**.

#### iOS

1. На устройстве/симуляторе открой **Safari** и перейди по адресу из Charles (обычно `chls.pro/ssl`).
2. Установи профиль (появится диалог установки).
3. Перейди в **Настройки → Основные → Об этом устройстве → Доверие сертификатов** и **включи** доверие для Charles Proxy CA.

> 📌 **Важно:** на iOS нужно выполнить **оба** шага: установить профиль **и** включить доверие. Без второго шага HTTPS не расшифруется.

---

### Шаг 4: Настройка SSL Proxying в Charles

1. В Charles: **Proxy → SSL Proxying Settings**.
2. Поставь галочку **Enable SSL Proxying**.
3. В списке **Include** нажми **Add** и добавь:

| Host | Port |
|------|------|
| `*` | `443` |

> 💡 Можно указать конкретные хосты вместо `*` для более точечного перехвата:
> - `api.exchangerate.host`
> - `open.er-api.com`

4. Нажми **OK**.

---

### Шаг 5: Запуск FinControl и проверка трафика

1. Убедись, что Charles запущен и запись включена: **Proxy → Start Recording** (или кнопка с красным кружком на панели).
2. На устройстве/эмуляторе открой приложение **FinControl**.
3. Перейди на экран **Обменник** или на **главный экран** (где подгружаются курсы валют).
4. Переключись в Charles — в дереве запросов (левая панель, режим **Structure**) должны появиться:
   - `api.exchangerate.host` — запрос вида `https://api.exchangerate.host/latest?base=RUB&symbols=USD,EUR,GBP,CHF,CNY,JPY,KZT,TRY,BRL,INR`
   - или `open.er-api.com` — запрос вида `https://open.er-api.com/v6/latest/RUB` (если основной API недоступен)
5. Кликни на запрос → вкладка **Response** → **JSON Text** — ты увидишь ответ с полями `rates`, `base`, `date`.

> 📌 Если курсы уже закэшированы и запроса нет — в **Настройках** приложения FinControl нажми **«Очистить данные»**, затем снова открой экран Обменник.

## 🛠 Траблшутинг

| Проблема | Решение |
|----------|---------|
| **Трафик не появляется в Charles** | Проверь пошагово: прокси на устройстве (IP и порт 8888), телефон и ПК в одной Wi-Fi сети, сертификат установлен и доверен, SSL Proxying включён, приложение реально запрашивает курсы (открой Обменник, при необходимости очисти кэш). Подробнее: [FAQ](../FAQ.md#charles--proxyman-не-видит-трафик-приложения-что-проверить) |
| **Ответ «Unknown» или не расшифрован** | Сертификат не установлен или не помечен как доверенный. Переустанови сертификат и проверь раздел «Доверенные учётные данные». Подробнее: [FAQ](../FAQ.md#в-charles-вижу-запрос-но-ответ-unknown-или-не-расшифрован) |
| **Порт 8888 занят** | Измени порт в **Proxy → Proxy Settings** (например на 8889) и обнови настройки прокси на устройстве. Подробнее: [FAQ](../FAQ.md#порт-8888-charles-занят-или-address-already-in-use) |
| **Веб-сборка не видна** | Настрой прокси браузера на `localhost:8888` и установи сертификат Charles в браузере (Charles подскажет URL) |

## 🧪 Практические сценарии тестирования

Ниже — конкретные сценарии тестирования приложения FinControl через Charles. Каждый сценарий описан пошагово: что делать в Charles, что делать в приложении, что проверять.

---

### Сценарий 1: Проверка корректности запроса к API курсов

**Цель:** убедиться, что приложение отправляет правильный запрос к API.

1. Запусти Charles, включи запись.
2. Открой FinControl → экран **Обменник**.
3. В Charles найди запрос к `api.exchangerate.host`.
4. **Проверь запрос:**
   - Метод: `GET`
   - URL: `https://api.exchangerate.host/latest?base=RUB&symbols=USD,EUR,GBP,CHF,CNY,JPY,KZT,TRY,BRL,INR`
   - Заголовки: `Accept: application/json`, `User-Agent: fin_control/1.0`
5. **Проверь ответ:**
   - Статус: `200`
   - Content-Type: `application/json`
   - Тело: JSON с полем `rates` (объект с курсами валют)
6. **Запиши:** какие валюты вернулись, совпадают ли они с отображёнными в приложении.

---

### Сценарий 2: Подмена курса валюты через Breakpoint

**Цель:** проверить, как приложение отображает подменённый курс (валидация данных с API).

1. В Charles: **Proxy → Breakpoint Settings** → **Enable Breakpoints** → **Add**:
   - **Host**: `api.exchangerate.host`
   - **Path**: `*`
   - **Response**: включить (ставим галочку)
2. В приложении нажми **Обновить** курсы (или перейди на Обменник).
3. Charles остановит ответ — откроется окно **Edit Response**.
4. В теле JSON найди `"USD":0.011...` и измени на `"USD":999.99`.
5. Нажми **Execute**.
6. **Проверь в приложении:** курс USD отображается как **999.99** (или близко к этому значению). Это доказывает, что приложение доверяет данным API без дополнительной валидации.
7. **Вывод для баг-репорта:** приложение не проверяет адекватность курсов — при подмене API может отображать некорректные данные.

---

### Сценарий 3: Имитация ошибки сервера (HTTP 500)

**Цель:** проверить обработку ошибок приложением и fallback-механизм.

1. В Charles: установи **Breakpoint** на ответ от `api.exchangerate.host` (как в сценарии 2).
2. В приложении обнови курсы.
3. Charles остановит ответ — измени **HTTP Status** на `500 Internal Server Error`.
4. Нажми **Execute**.
5. **Проверь в приложении:**
   - Приложение должно попытаться использовать fallback-провайдер (`open.er-api.com`)
   - Если оба провайдера недоступны — должно показать кэшированные курсы или сообщение об ошибке
6. **Что записать:** переключилось ли приложение на fallback? Показало ли ошибку? Не упало ли?

---

### Сценарий 4: Невалидный JSON в ответе

**Цель:** проверить устойчивость парсера к некорректным данным.

1. Установи **Breakpoint** на ответ API курсов.
2. В приложении обнови курсы.
3. В окне **Edit Response** замени тело на:
   ```json
   {"error": "invalid"}
   ```
   (валидный JSON, но без поля `rates`).
4. **Проверь:** приложение не упало? Показало ошибку или переключилось на fallback?
5. Повтори с телом `not a json at all` (полностью невалидный JSON).
6. **Вывод:** приложение должно корректно обрабатывать оба случая — и отсутствие нужных полей, и битый формат.

---

### Сценарий 5: Проверка поведения при медленной сети (Throttling)

**Цель:** проверить таймаут и UX при плохом интернете.

1. В Charles: **Proxy → Throttle Settings** → **Enable Throttling**.
2. Выбери пресет **56 kbps Modem** или задай **Latency: 5000 ms**.
3. В разделе **Only for selected hosts** добавь `api.exchangerate.host`.
4. В приложении обнови курсы.
5. **Проверь:**
   - Показывается ли индикатор загрузки (спиннер)?
   - Не зависает ли UI (можно ли переключать вкладки)?
   - Срабатывает ли таймаут (5 секунд в FinControl)?
   - При таймауте — показываются ли кэшированные курсы?
6. **Выключи Throttling** после теста.

---

### Сценарий 6: Проверка обмена валюты (локальная операция)

**Цель:** убедиться, что обмен валюты — локальная операция и не отправляет данные на сервер.

1. Включи запись в Charles.
2. В приложении: экран **Обменник** → введи сумму `5000` → выбери **RUB → USD** → нажми **Обменять**.
3. В Charles проверь: **не должно** появиться дополнительных запросов при обмене (обмен — локальная операция, на API не ходит).
4. В приложении проверь: операция появилась в **Истории обменов** с правильными суммами и курсом.
5. **Вывод:** обмен — локальная операция; трафик к API идёт только при загрузке курсов.

---

### Сценарий 7: Map Local — подготовка фиксированных курсов для тестирования

**Цель:** создать предсказуемые условия для тестирования с заранее известными данными.

1. Создай файл `test_rates.json` на компьютере:
   ```json
   {
     "motd": {"msg": "test"},
     "success": true,
     "base": "RUB",
     "date": "2026-01-15",
     "rates": {
       "USD": 0.01,
       "EUR": 0.009,
       "GBP": 0.008,
       "CHF": 0.0095,
       "CNY": 0.075,
       "JPY": 1.55,
       "KZT": 5.0,
       "TRY": 0.35,
       "BRL": 0.055,
       "INR": 0.85
     }
   }
   ```
2. В Charles: **Tools → Map Local** → **Add**:
   - **Host**: `api.exchangerate.host`
   - **Path**: `/latest*`
   - **Local path**: путь к файлу `test_rates.json`
3. В приложении обнови курсы — они загрузятся из твоего файла.
4. Теперь ты контролируешь данные: можно тестировать обмен, портфель, покупку акций с предсказуемыми курсами.
5. **Для баг-хантинга** — попробуй экстремальные значения в файле:

| Значение в JSON | Что проверяем |
|-----------------|---------------|
| `"USD": 0` | Деление на ноль? |
| `"USD": -1` | Отрицательный курс? |
| `"USD": 999999999` | Переполнение в UI? |

## 🔧 Дополнительно: продвинутые функции Charles

### Breakpoints — изменение запроса/ответа на лету

Breakpoints позволяют остановить запрос или ответ, изменить его и отправить дальше. Это **ключевой навык** для тестирования.

1. **Proxy → Breakpoint Settings** → **Enable Breakpoints**.
2. Добавь правило: **Host** = `api.exchangerate.host`, **Path** = `*`, **Query** = `*`.
3. Выбери, на чём останавливаться: **Request** (перед отправкой на сервер) и/или **Response** (перед отдачей в приложение).
4. В приложении открой Обменник — запрос остановится.
5. В окне Breakpoint:
   - **Для Response:** измени тело JSON — например `"USD": 999.99`. Нажми **Execute**.
   - **Для Request:** измени параметры запроса — например `symbols=USD,EUR,GBP` → `symbols=USD`.
6. Проверь результат в приложении.

### Map Remote — подмена URL

Map Remote перенаправляет запросы с одного URL на другой. Полезно для тестирования с другим API или мок-сервером.

1. **Tools → Map Remote** → **Enable Map Remote** → **Add**.
2. **Map From:** `https://api.exchangerate.host/latest*`
3. **Map To:** `https://open.er-api.com/v6/latest/RUB` (или свой мок-сервер).
4. Все запросы к основному API будут уходить на fallback — проверь, что приложение работает корректно.

### Map Local — подмена ответа файлом

Map Local возвращает ответ из локального файла вместо реального запроса. Идеально для воспроизведения конкретных сценариев.

1. Создай файл `mock_rates.json`:
   ```json
   {"result":"success","base_code":"RUB","rates":{"USD":0.011,"EUR":0.0095,"GBP":0.008}}
   ```
2. **Tools → Map Local** → **Enable Map Local** → **Add**.
3. **Host:** `api.exchangerate.host`, **Path:** `/latest*`.
4. **Local path:** путь к `mock_rates.json`.
5. Открой приложение — курсы загрузятся из файла.

**Сценарии для тестирования через Map Local:**
- Файл с экстремальными курсами (0.0001, 99999.99) — проверь отображение
- Пустой ответ `{}` — проверь обработку
- Ответ без обязательных полей — проверь устойчивость парсера

### Rewrite — автоматическая модификация

Rewrite автоматически модифицирует запросы/ответы по правилам (без остановки, в отличие от Breakpoints).

1. **Tools → Rewrite** → **Enable Rewrite** → **Add**.
2. Добавь **Location**: `api.exchangerate.host`.
3. Добавь правило:
   - **Type:** Body → **Match:** `"USD":` → **Replace:** `"USD":0.001,` — все ответы будут содержать изменённый курс USD
   - **Type:** Header → добавить заголовок `X-Test: true` ко всем запросам

### Throttling — имитация медленной сети

1. **Proxy → Throttle Settings** → **Enable Throttling**.
2. Выбери пресет:
   - **56 kbps Modem** — очень медленная сеть
   - **3G** — типичная мобильная сеть
   - **Custom** — свои значения Bandwidth, Latency, Reliability
3. Включи **Only for selected hosts** → добавь `api.exchangerate.host`.
4. В приложении открой Обменник и проверь: индикатор загрузки, отзывчивость UI, таймаут, кэш.

### Repeat & Repeat Advanced

- **Правый клик на запросе → Repeat** — повторить запрос один раз.
- **Repeat Advanced** — повторить N раз с задержкой (нагрузочное тестирование).

> 💡 Пример: повтори запрос курсов 50 раз с задержкой 100 мс — проверь, что API не блокирует и приложение корректно обрабатывает все ответы.

### No Caching

**Tools → No Caching** — отключает кэширование на стороне клиента (убирает заголовки `If-Modified-Since`, `If-None-Match`). Полезно, чтобы каждый запрос возвращал свежие данные.

### Compose — создание запроса вручную

**Tools → Compose** — создай HTTP-запрос с нуля и отправь на сервер. Полезно для тестирования API без приложения:

1. **Method:** `GET`
2. **URL:** `https://api.exchangerate.host/latest?base=RUB&symbols=USD,EUR`
3. Добавь заголовки: `Accept: application/json`, `User-Agent: fin_control/1.0`
4. **Execute** — увидишь ответ прямо в Charles.
