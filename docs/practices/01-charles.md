# Практика: Charles Proxy — перехват трафика FinControl

## Цель

Настроить Charles Proxy и увидеть HTTP/HTTPS-запросы приложения FinControl к API курсов валют (мобильное и при необходимости веб-приложение).

## Что понадобится

- Charles Proxy ([charlesproxy.com](https://www.charlesproxy.com/))
- Устройство Android (эмулятор или телефон) или iOS-симулятор/устройство
- Приложение FinControl (собранное и установленное)

## Шаг 1: Установка Charles

1. Скачайте Charles с официального сайта, установите.
2. Запустите Charles. При первом запуске разрешите установку сертификата Charles (для расшифровки HTTPS).

## Шаг 2: Настройка прокси на устройстве/эмуляторе

### Android-эмулятор

1. В эмуляторе: **Settings → Network & internet → Internet** (или **Proxy** в настройках Wi‑Fi).
2. Укажите ручной прокси: **Host** — IP вашего компьютера в локальной сети (например `192.168.1.100`), **Port** — обычно `8888` (порт Charles по умолчанию).
3. Альтернатива: в AVD можно задать `-http-proxy http://<ваш_IP>:8888` при запуске эмулятора.

### Реальное Android-устройство

1. Убедитесь, что телефон и компьютер в одной Wi‑Fi сети.
2. В Charles: **Proxy → Proxy Settings** — порт 8888.
3. На телефоне: **Wi‑Fi → долгое нажатие по сети → Изменить / Дополнительно → Прокси вручную** — укажите IP компьютера и порт 8888.

### iOS-симулятор

Симулятор использует сеть Mac/ПК. В Charles настройте **Proxy → macOS Proxy** (или Windows Proxy), чтобы перехватывать трафик приложений на том же компьютере.

## Шаг 3: Установка сертификата Charles на устройстве (HTTPS)

Без доверенного сертификата Charles вы не увидите расшифрованный HTTPS.

### Android

1. В Charles: **Help → SSL Proxying → Install Charles Root Certificate on a Mobile Device or Remote Browser**.
2. Откройте на устройстве браузер и перейдите по указанному адресу (например `chls.pro/ssl`).
3. Скачайте и установите сертификат. В Android 7+ для приложений может потребоваться поместить сертификат в «Доверенные учётные данные» (Settings → Security → Encryption & credentials → Install a certificate).

### iOS

1. На устройстве/симуляторе откройте в Safari адрес из Charles (**Help → SSL Proxying → Install Charles Root Certificate…**).
2. Установите профиль, затем **Настройки → Основные → Об этом устройстве → Доверие** — включите доверие для Charles.

## Шаг 4: SSL Proxying в Charles

1. **Proxy → SSL Proxying Settings**.
2. Включите **Enable SSL Proxying**, добавьте **Host** `*` и **Port** `443` (или конкретные хосты: `api.exchangerate.host`, `open.er-api.com`).

## Шаг 5: Запуск FinControl и проверка трафика

1. Запустите Charles, включите запись (**Proxy → Start Recording**).
2. На устройстве/эмуляторе откройте приложение FinControl.
3. Перейдите на экран **Обменник** или на главный экран (где подгружаются курсы валют).
4. В Charles в дереве запросов должны появиться вызовы к:
   - `api.exchangerate.host` (например `https://api.exchangerate.host/latest?base=RUB&symbols=USD,EUR`),
   - или `open.er-api.com` (fallback, например `https://open.er-api.com/v6/latest/RUB`).

## Что проверить (чек-лист для ученика)

- [ ] Charles запущен, прокси на устройстве указан, порт 8888.
- [ ] Сертификат Charles установлен и доверен на устройстве.
- [ ] SSL Proxying включён для нужных хостов.
- [ ] В FinControl открыт экран с загрузкой курсов (Обменник или главный с карточкой курсов).
- [ ] В Charles виден запрос к `exchangerate.host` или `open.er-api.com`, ответ 200, тело JSON с полями `rates`, `date` или аналог.

## Устранение неполадок

- **Нет трафика в Charles**: проверьте, что прокси на устройстве указывает на IP компьютера с Charles и что приложение действительно делает сетевой запрос (включён интернет, нет кэша).
- **SSL handshake failed**: установите и доверьте сертификат Charles на устройстве, перезапустите приложение.
- **Только веб**: если тестируете веб-сборку FinControl в браузере, настройте прокси браузера на `localhost:8888` и установите сертификат Charles в браузере.

## Что показать на экзамене / созвоне

1. Запустить Charles, убедиться, что запись включена.
2. На устройстве/эмуляторе открыть FinControl → экран **Обменник** (или главный с карточкой курсов).
3. В Charles показать дерево запросов: есть запрос к `api.exchangerate.host` или `open.er-api.com`.
4. Открыть ответ: HTTP 200, в теле JSON с полями курсов (rates).
5. Кратко сказать: «Настроил прокси на устройстве, установил и доверил сертификат Charles. При открытии курсов в приложении запрос к API виден в Charles, HTTPS расшифровывается.»

## Дополнительно

- Использование **Breakpoints** в Charles для изменения запроса/ответа.
- **Throttling** для проверки поведения при медленной сети.
