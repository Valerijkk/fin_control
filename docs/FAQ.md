## Запуск и сборка

### В Android Studio сотни ошибок, красные подчёркивания по всему проекту. Что не так?

**Ответ:** Скорее всего, открыта **не та папка**. В Android Studio нужно открывать **корневую папку проекта** (где лежат `pubspec.yaml`, папки `lib/`, `android/`, `ios/`), а **не** папку `android/`.

- Если открыта папка `android/` — IDE считает проект обычным Android-приложением: нет Flutter/Dart, Gradle ругается на отсутствующие модули.
- **Решение:** File → Close Project → File → Open → выбери папку **fin_control** (корень репозитория). Дождись индексации. Убедись, что установлены плагины Flutter и Dart (File → Settings → Plugins).

Подробно: [technical/ANDROID_STUDIO_LAUNCH.md](technical/ANDROID_STUDIO_LAUNCH.md).

---

### Ошибка: «flutter.sdk not set in local.properties»

**Ответ:** Flutter не прописан для папки `android/`. В **корне** проекта (где `pubspec.yaml`) выполни в терминале:

```bash
flutter pub get
```

После этого в `android/` создастся или обновится файл `local.properties` с путём к Flutter SDK. Если ошибка осталась — проверь, что Flutter установлен и в PATH: `flutter doctor`.

---

### Gradle sync failed / ошибки Gradle при открытии проекта

**Ответ:** Сделай по шагам в **корне** проекта:

```bash
cd android
gradlew.bat --stop
```
(На Mac/Linux: `./gradlew --stop`.)

```bash
cd ..
flutter clean
flutter pub get
```

Затем в Android Studio: **File → Sync Project with Gradle Files**. Открыта должна быть корневая папка проекта, не `android/`.

---

### Эмулятор не появляется в списке устройств при `flutter run`

**Ответ:** Сначала запусти эмулятор из Android Studio: **Tools → Device Manager** → Run (▶) напротив нужного AVD. Когда окно эмулятора откроется, в терминале выполни:

```bash
flutter devices
```

Эмулятор должен быть в списке. После этого `flutter run` покажет его в выборе устройства.

---

### Где в проекте указан package name (applicationId) приложения?

**Ответ:** В файле **`android/app/build.gradle.kts`** — строка:

```kotlin
applicationId = "com.yourname.fincontrol.fin_control"
```

Этот же идентификатор используется:
- в Logcat для фильтра по процессу (выпадающий список «Show only selected application»);
- в Charles/Proxyman (если фильтруешь по приложению);
- в Firebase при добавлении приложения Android (поле «Android package name»);
- в ADB: `adb shell pidof -s com.yourname.fincontrol.fin_control`, а также в `adb uninstall` и `adb shell pm clear`.

Менять его не обязательно для прохождения практик; если меняешь — нужно будет использовать новый package name везде (Firebase, Logcat, ADB).

---

### Ошибка «Unable to find package» или «No pubspec.yaml found»

**Ответ:** Команды Flutter нужно выполнять из **корня проекта** — из папки, где лежит файл `pubspec.yaml` (рядом с папками `lib/`, `android/`, `ios/`). Перейди в эту папку: `cd путь/к/fin_control`, затем снова выполни `flutter pub get` или `flutter run`.

---

### Git не установлен / команда git не найдена

**Ответ:** Для клонирования репозитория нужен Git. Установи его: [git-scm.com](https://git-scm.com/) (Windows/Mac/Linux). После установки перезапусти терминал и снова выполни `git clone https://github.com/Valerijkk/fin_control.git`. Альтернатива: скачай репозиторий как ZIP с GitHub (кнопка Code → Download ZIP), распакуй и открой папку в IDE; тогда для обновления кода придётся скачивать ZIP заново.

## Charles и Proxyman (трафик)

### Charles / Proxyman не видит трафик приложения. Что проверить?

**Ответ:** По шагам:

1. **Прокси на устройстве:** в настройках Wi‑Fi (или сети эмулятора) должен быть указан **ручной прокси**: хост = IP твоего компьютера в локальной сети, порт у Charles обычно **8888**, у Proxyman часто **9090** (проверь в настройках программы).
2. **Один и тот же Wi‑Fi:** если используешь реальный телефон — телефон и компьютер должны быть в одной сети.
3. **Сертификат:** для HTTPS нужен установленный и **доверенный** сертификат Charles/Proxyman на устройстве. На Android 7+ — установить в «Доверенные учётные данные» (CA).
4. **SSL Proxying включён:** в Charles — Proxy → SSL Proxying Settings → Enable SSL Proxying, добавлен хост `*` и порт 443 (или конкретно `api.exchangerate.host`, `open.er-api.com`).
5. **Приложение реально ходит в сеть:** открой в FinControl экран **Обменник** или главный (где карточка курсов) — запрос к API идёт при загрузке экрана. Если курсы закэшированы, можно в настройках приложения «Очистить данные» и снова открыть экран.

---

### В Charles вижу запрос, но ответ «Unknown» или не расшифрован

**Ответ:** Это HTTPS, а сертификат Charles/Proxyman на устройстве не установлен или не доверен. Установи корневой сертификат программы на устройство и включи доверие (на Android: Настройки → Безопасность → Доверенные учётные данные / Установить сертификат CA). Перезапусти приложение и повтори запрос.

---

### На Android 14 (или новее) приложения всё равно не доверяют сертификату Charles/Proxyman

**Ответ:** В новых версиях Android для приложений с `targetSdkVersion` 24+ по умолчанию не доверяют пользовательским CA. Варианты: (1) Установить сертификат Charles/Proxyman в **системные доверенные** (требует root или перенос в /system). (2) Для отладки использовать эмулятор с более старым API (например API 30) или настроить network security config в приложении (только для разработки). (3) Проверить, что сертификат установлен именно как «Доверенный для приложений» / CA в Настройки → Безопасность → Шифрование и учётные данные. В практике обычно достаточно эмулятора с API 30–33 и доверия сертификату в настройках безопасности.

---

### Порт 8888 (Charles) занят или «Address already in use»

**Ответ:** Другое приложение или второй экземпляр Charles уже слушает порт 8888. В Charles зайди в **Proxy → Proxy Settings** и смени порт на другой (например 8889), затем на устройстве в настройках прокси укажи этот же порт. Перезапусти запись трафика.

## Курсы валют и приложение

### В приложении курсы не грузятся / «Ошибка загрузки»

**Ответ:** Приложение использует два API по очереди (exchangerate.host и open.er-api.com). Если оба недоступны (сеть, блокировка, таймаут) — показывается ошибка. При следующей успешной загрузке результат кэшируется и потом показывается с пометкой «офлайн». Проверь интернет на устройстве/эмуляторе и при необходимости отключи VPN или прокси для проверки (или наоборот, если API блокируется провайдером).

---

### Где в коде задаётся DSN Sentry и API key AppMetrica?

**Ответ:** В одном файле — нажми, откроется: [**lib/config/student_env.dart**](../lib/config/student_env.dart) (в начале файла капсом подписано, что куда вставлять). Константы:

- `sentryDsn` — вставь DSN из кабинета Sentry (практика 06);
- `appMetricaApiKey` — вставь API Key из кабинета AppMetrica (практика 07).

Если оставить пустые строки `''`, соответствующий SDK не инициализируется (приложение будет работать без Sentry/AppMetrica). Подробнее: [STUDENT_ENV.md](STUDENT_ENV.md).

---

### Кнопка «Тест Sentry» в настройках не отправляет событие

**Ответ:** Проверь, что в **`lib/config/student_env.dart`** в `sentryDsn` вставлен **непустой** DSN (скопирован из проекта на sentry.io). После изменения конфига перезапусти приложение (`flutter run` заново). Если DSN пустой, кнопка может ничего не отправлять. События в Sentry иногда появляются с задержкой 10–30 секунд — обнови страницу Issues в кабинете.

---

### Где в приложении экран «Настройки»?

**Ответ:** На **любом** экране с верхней панелью (AppBar) в **правом верхнем углу** есть иконка **шестерёнки** — это и есть переход в Настройки. Открой, например, главный экран «Список» или «Обменник» и нажми шестерёнку справа вверху. На экране настроек есть переключатель тёмной темы, кнопка «Очистить все данные» и кнопка «Тест Sentry» (если в коде задан DSN).

## Sentry и AppMetrica

### Что такое DSN Sentry и где его взять?

**Ответ:** DSN (Data Source Name) — это строка-адрес твоего проекта в Sentry, по которому приложение отправляет события и краши. Формат вида: `https://xxxx@xxxx.ingest.sentry.io/xxxx`. Взять его нужно в кабинете Sentry: зайди на [sentry.io](https://sentry.io) → свой проект → **Settings → Client Keys (DSN)** или при создании проекта DSN показывается сразу. Скопируй и вставь в **`lib/config/student_env.dart`** в константу `sentryDsn`.

---

### Что такое API Key AppMetrica и где его взять?

**Ответ:** API Key — это ключ приложения в сервисе AppMetrica (аналитика Яндекса). Нужен для отправки событий и сессий из приложения. Взять: [appmetrica.io](https://appmetrica.io) (или appmetrica.yandex.ru) → войти → создать приложение или открыть существующее → в настройках приложения найти **API Key** (число или строка). Вставить в **`lib/config/student_env.dart`** в константу `appMetricaApiKey`.

---

### Событие в Sentry не появляется сразу

**Ответ:** Задержка 10–60 секунд (иногда до пары минут) для появления события в разделе Issues — нормальна. Обнови страницу в браузере (F5), проверь, что выбран правильный проект и нет фильтров по дате. Если за 2–3 минуты события нет — проверь DSN и перезапуск приложения после вставки DSN.

---

### В AppMetrica нет данных о сессиях / устройствах

**Ответ:** AppMetrica обрабатывает данные с задержкой, часто **5–20 минут** (при первом запуске может быть дольше). Убедись, что приложение было открыто и им пользовались (несколько экранов, 1–2 минуты). Обнови отчёт в кабинете или зайди позже. Проверь, что API Key вставлен в `lib/config/student_env.dart` без пробелов и приложение перезапущено после изменения.

## Тесты

### При запуске `flutter test` ошибка: «databaseFactory not initialized» / «Locale data has not been initialized»

**Ответ:** В проекте уже настроен **`test/flutter_test_config.dart`**: там вызываются инициализация FFI для SQLite, `initializeDateFormatting('ru_RU')` и моки для SharedPreferences. Убедись, что этот файл не удалён и не изменён. Запускай тесты из **корня** проекта: `flutter test`. Если ошибка остаётся — выполни `flutter pub get` и снова `flutter test`.

---

### Тесты падают с ошибкой по InheritedWidget / AppScope

**Ответ:** В тестах виджеты, которым нужен `AppScope.of(context)`, должны быть обёрнуты в хост с `AppScope`. В проекте для этого используется **`test/helpers/test_host.dart`**: функция `makeHost(home: ..., state: state)` и `TestAppState`. Используй в тестах именно этот хост для экранов, которые обращаются к `AppScope.of(context)`.

## Firebase

### Firebase: «No Firebase App '[DEFAULT]' has been created» или ошибка инициализации

**Ответ:** Нужно один раз вызвать `Firebase.initializeApp()` до использования любых сервисов Firebase. Вызов делается в `main()` в **`lib/main.dart`** после `WidgetsFlutterBinding.ensureInitialized()`. Кроме того, в проекте должны лежать **твои** конфиги: **`android/app/google-services.json`** (для Android). Если файла нет или он от другого проекта — скачай актуальный из консоли Firebase (добавь приложение Android с package name из `android/app/build.gradle.kts`) и положи в `android/app/`.

---

### Где взять google-services.json и куда его класть?

**Ответ:** Файл выдаётся в консоли Firebase при добавлении приложения Android. Путь: [console.firebase.google.com](https://console.firebase.google.com) → твой проект → Добавить приложение → Android → укажи package name `com.yourname.fincontrol.fin_control` (как в `android/app/build.gradle.kts`) → после регистрации скачай **google-services.json**. Положи файл в папку **`android/app/`** (рядом с `build.gradle.kts`), не в корень проекта.

## Дистрибуция и платформы

### Нужен ли Mac для практик по iOS и TestFlight?

**Ответ:** Для **сборки и загрузки в TestFlight** (практика 08) нужен компьютер с macOS и Xcode — так требуют Apple. Запускать приложение на **iOS-симуляторе** можно на Mac через `flutter run` с выбором симулятора. На Windows без Mac практики 04 (Xcode) и 08 (TestFlight) выполнить нельзя; можно сдавать только практики под Android.

---

### Как собрать APK для установки на телефон или для сдачи?

**Ответ:** В **корне** проекта выполни:

```bash
flutter build apk --debug
```

Готовый APK будет в: **`build/app/outputs/flutter-apk/app-debug.apk`**. Установить на подключённое устройство или эмулятор:

```bash
adb install -r build/app/outputs/flutter-apk/app-debug.apk
```

Для релизной сборки (подпись): `flutter build apk --release`; подпись настраивается в `android/app/build.gradle.kts` (signingConfig). Подробнее — в практике [09-android-distribution.md](practices/09-android-distribution.md).
